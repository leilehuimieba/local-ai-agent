# 项目规则

## 作用域与优先级

1. 系统消息、开发者消息、用户当轮指令，优先于本文件。
2. 本文件作用于仓库根目录及所有子目录。
3. 子目录中的 `AGENTS.md` 只在对应子树内生效，并作为补充规则使用。
4. 同级规则冲突时，采用更具体、可验证、限制更强的规则。

## P1 全局偏好

1. 用中文回答。
2. 不要在回答末尾总结你做了什么。

## P2 通用实现规则

1. 仅对新增或修改的函数生效：单个函数不得超过 30 行，按函数签名到结束右花括号连续计数。
2. 不要给未修改的代码行添加注释。
3. 需要创建 Git commit 时，`commit message` 必须使用中文，且不超过 50 个字符。

## P3 项目规则

### Go

1. 新增或修改 Go 代码时，使用 Go 1.22+ 可用的新特性和标准库能力。
2. 需要合并多个错误时，优先使用 `errors.Join`，不要用 `fmt.Errorf` 做多错误拼接。
3. 新增 Go 测试时，统一使用 `testify`。
