# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 当前前端已具备工作台雏形，但整体仍偏“页面集合”，导航、任务主线程、事件流、日志、设置之间的视觉与交互范式尚未完全统一。
  2. 现有前端在信息架构上已经拆出 `shell / workspace / chat / events / logs / settings / history / resources`，但用户仍需要分别理解多块 UI 的语义，学习成本高于成熟 AI agent 工作台。
  3. 参考 Codex 一类成熟桌面端 AI 编程助手的工作台范式，可以减少用户的重新学习成本，并减少后续新增能力时的重复设计消耗。
- 对应阶段目标：
  1. 阶段 H（产品差异化与透明执行）：在不改写后端 contract 的前提下，收敛前端为更透明、更一致、更易理解的 agent workbench。

## 目标

- 本次要完成什么：
  1. 将当前前端重构为统一的 AI agent 工作台壳层，对齐 Codex 风格的“导航 + 主任务流 + 调查/日志/输入器”结构。
  2. 建立可持续扩展的前端设计基础，包括深色工作台 token、统一 panel/card/status 语义和最小 UI primitives 约束。
  3. 输出“现状模块 -> 目标模块”的映射方案，明确哪些模块先做 shell 收口，哪些模块后做内容块收口。
  4. 在不切换当前主推进 change 的前提下，提供一个可独立推进的前端重构工作区，供后续逐步实现与验收。

## 非目标

- 本次明确不做什么：
  1. 不在本 change 中改写后端接口 contract、事件协议或 runtime 状态机。
  2. 不在本 change 中一次性重写全部前端业务组件和交互细节。
  3. 不追求像素级复制 Codex 客户端，只复用成熟工作台的信息架构与交互范式。
  4. 不在本 change 中改动 `current-state.md` 所代表的当前主推进项；当前主推进仍保持 `H-gate-h-signoff-20260416`。

## 验收口径

- 通过标准：
  1. `proposal/design/tasks/status/verify` 五件套齐备，并已加入 `changes/INDEX.md` 作为并行 change。
  2. 已形成清晰的前端重构边界：哪些属于 shell/token 重构，哪些属于聊天/事件/日志内容块重构，哪些后置。
  3. 已形成可执行的模块映射表，能够明确指导后续实现顺序与影响范围。
  4. 已定义最小验证口径，能判断这次重构是否真正降低学习成本并减少后续重复设计投入。
