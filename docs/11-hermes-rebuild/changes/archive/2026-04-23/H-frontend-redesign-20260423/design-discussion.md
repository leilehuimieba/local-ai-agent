## 一、当前前端问题诊断

### 1.1 视觉层面

| 问题 | 表现 | 影响 |
|------|------|------|
| TopBar 过载 | 品牌 + 导航 + 6 个状态指标 + 2 个按钮挤在一行 | 视线无处落脚，重要信息被稀释 |
| Sidebar 堆砌 | 5-6 个卡片垂直堆叠，每个卡片内又有 3-7 行标签-值对 | 信息密度过高，无重点 |
| 卡片同质化 | 所有卡片使用相同灰度背景，缺乏视觉区分 | 用户无法快速识别模块边界 |
| 输入框突兀 | 底部白色输入框与深色主题完全不协调 | 视觉断裂感强 |
| 标签噪音 | "工作台模块"、"工作台侧栏"等标签在每个卡片重复 | 大量无信息增量的视觉噪音 |

### 1.2 布局层面

| 问题 | 表现 | 影响 |
|------|------|------|
| 三栏失衡 | 顶部(header)占太多空间，中间内容区被挤压 | 核心任务区空间不足 |
| 信息重复 | "当前任务"在 Overview 和 TaskGroup 同时出现 | 同样信息看两次 |
| 空状态占位 | 大量"当前没有..."的占位文本占据空间 | 界面始终显得拥挤 |
| 碎片化 | 首页左侧拆成 3-4 个不同尺寸卡片 | 视线需要不断跳跃 |

### 1.3 信息架构层面

| 问题 | 表现 | 影响 |
|------|------|------|
| 状态分散 | 运行状态在 TopBar、Sidebar、BottomPanel 都有展示 | 不知道哪个是权威状态 |
| 语义混杂 | "任务工作区"和"任务主线程"边界不清 | 用户困惑自己在哪里 |
| 导航弱 | 只有顶部 4 个文字按钮，无图标辅助 | 切换视图不够直觉 |

## 二、Dribbble 2026 设计趋势参考

基于对 dribbble 近期高赞作品的分析，总结出以下可用趋势：

### 2.1 布局趋势

1. **左侧垂直导航**：图标 + 文字，当前项高亮，可收起。释放顶部空间。
2. **中心聚焦**：主内容区只放一个核心任务，次要信息放到可折叠面板。
3. **右侧详情面板**：仅在需要时展开，平时隐藏或很窄。
4. **大面积留白**：模块之间用留白分隔，减少边框使用。

### 2.2 视觉趋势

1. **克制的强调色**：深色背景上只用 1-2 种品牌色，状态色极度收敛。
2. **圆角统一**：大圆角(16px)用于卡片，小圆角(8px)用于按钮/标签。
3. **微阴影代替边框**：用微弱的背景色差异和阴影区分层级，减少硬边框。
4. **字体层级严格**：标题/正文/辅助文字只用 3-4 个尺寸，差异明显。

### 2.3 交互趋势

1. **渐进披露**：默认只展示最重要的，点击/悬浮再展开详情。
2. **状态可视化**：用颜色条、进度环、脉冲动画代替文字状态。
3. **上下文感知**：根据当前状态自动调整界面，空状态时界面极简。

## 三、新设计方向提案

### 3.1 核心原则

1. **少即是多**：默认界面只展示用户当前最需要的信息。
2. **一层一焦点**：每个视图只有一个视觉焦点区域。
3. **状态可视化**：能用颜色/图标/进度表达的不用文字。
4. **空间换 clarity**：用留白和呼吸感换信息清晰度。

### 3.2 布局重构方案

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo]  本地智能体                              [状态指示灯] │  ← 极简顶部栏
├────────┬──────────────────────────────────────────┬─────────┤
│        │                                          │         │
│  🏠    │                                          │   📊    │
│  💬    │         主内容区（单焦点）                │  状态   │
│  📋    │                                          │  摘要   │
│  ⚙️    │                                          │         │
│        │                                          │         │
│        │                                          │         │
├────────┴──────────────────────────────────────────┴─────────┤
│  [输入任务，按回车发送...]                              [发送] │  ← 底部输入区（融合主题）
└─────────────────────────────────────────────────────────────┘
```

**变化说明：**
- 顶部：只保留 Logo + 全局状态指示灯（颜色编码），导航移到左侧。
- 左侧：图标导航栏（首页/任务/记录/设置），当前页高亮。
- 中间：单一大块主内容区，根据当前视图展示核心内容。
- 右侧：可选的窄面板，仅在任务/记录页展示关键状态摘要。
- 底部：融合主题的输入条，不再用突兀的白色。

### 3.3 各视图简化方案

**首页（Home）**
- 当前：6-7 个卡片堆砌
- 新方案：
  - 大标题 + 一句话欢迎
  - 一个显眼的输入框（占据视觉中心）
  - 下方 2-3 个快速操作卡片（最近任务/新建任务/设置引导）
  - 空状态时只显示欢迎 + 输入框，极简

**任务页（Task）**
- 当前：任务工作区 + 任务主线程 + 底部调查层 + 右侧sidebar
- 新方案：
  - 主区：任务对话流（类似简洁的聊天界面，但非气泡式）
  - 右侧窄面板：当前状态摘要（3-4 个关键指标）
  - 底部：可折叠的调查轨道（默认收起，点击展开）

**记录页（Logs）**
- 当前：筛选台 + 时间线 + 焦点卡，信息密度过高
- 新方案：
  - 顶部简洁筛选栏（一行搞定）
  - 主区：时间线列表（精简版）
  - 点击后右侧展开详情

**设置页（Settings）**
- 当前：大量卡片网格排列
- 新方案：
  - 左侧子导航（运行时/模型/工作区/...）
  - 右侧表单区，减少卡片嵌套

### 3.4 Token 简化

当前 token 过多且差异过小，建议收敛：

```css
/* 背景层级 */
--bg-base: #0d0f12;        /* 最底层 */
--bg-surface: #141820;     /* 卡片/面板 */
--bg-elevated: #1c2130;    /* 悬浮/选中 */

/* 文字层级 */
--text-primary: #f0f2f5;   /* 标题/正文 */
--text-secondary: #8b92a8; /* 辅助文字 */
--text-muted: #5a6070;     /* 占位/禁用 */

/* 强调色 */
--accent: #5b8cff;
--accent-soft: rgba(91, 140, 255, 0.12);

/* 状态色（极度收敛） */
--status-success: #4ade80;
--status-warning: #fbbf24;
--status-error: #f87171;
--status-idle: #6b7280;

/* 边框 */
--border: rgba(255, 255, 255, 0.06);

/* 间距（4px 基准）*/
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-6: 24px;
--space-8: 32px;
--space-12: 48px;

/* 圆角 */
--radius-sm: 6px;
--radius-md: 10px;
--radius-lg: 16px;
```

## 四、待讨论事项

1. **导航位置**：左侧图标导航 vs 保留顶部导航？
2. **Sidebar 取舍**：右侧状态面板是常驻还是按需展开？
3. **深色主题**：当前深色是否太深？是否需要稍亮一点的深色（如 #1a1d29 作为 base）？
4. **强调色**：保持当前蓝色，还是考虑其他品牌色？
5. **实施节奏**：是渐进式改造（一次改一个视图）还是整体重写？
6. **参考产品**：除了 Dribbble，是否有具体的产品界面你特别喜欢？（如 Cursor、Claude Code、Warp 等）
