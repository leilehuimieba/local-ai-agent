# 知识库设计决策确认

更新时间：2026-04-23
状态：已确认

## 用户决策

1. **知识库和 Memory 彻底分开** ✅
   - Memory（记忆）：运行时自动沉淀，保留在 Settings 内或独立子页面
   - Knowledge Base（知识库）：用户主动管理的专业资料，独立导航项

2. **思源知识库可编辑** ✅
   - 不是只读展示
   - 前端编辑后同步回思源

3. **添加资料方式** ✅
   - 支持文件上传（PDF/Word/TXT/Markdown）
   - 支持输入网站 URL 提取信息（OpenCLI 辅助）
   - 支持手动输入 Markdown

4. **Obsidian 接入** ✅
   - 未来接入 Obsidian 作为另一知识库源
   - 复用思源接入模式（本地 Markdown 文件 → 摘要索引 → 正文读取）
   - 利用 Obsidian 图谱可视化能力

5. **实现节奏：方案 A** ✅
   - Wave 2：首页 + 任务页 + 记录页 + 设置页简化
   - Wave 3：知识库页面 + 记录页 + 设置页

## 知识库页面信息架构（冻结版）

### 核心概念：知识库源（Source）

一个知识库条目可以有不同来源：

| 来源 | 说明 | 编辑方式 |
|------|------|----------|
| `manual` | 用户在前端手动添加 | 前端直接编辑 |
| `siyuan` | 同步自思源笔记 | 前端编辑后回写思源 |
| `obsidian` | 同步自 Obsidian（未来） | 前端编辑后回写 Obsidian |
| `url` | 从网站 URL 提取 | 前端编辑后保存为本地副本 |
| `upload` | 从上传文件解析 | 前端编辑后保存为本地副本 |

### 页面布局

```
知识库页面（Knowledge Base Workspace）
├── 顶部操作栏
│   ├── 知识库切换器（默认知识库 / 思源知识库 / ...）
│   ├── [+ 添加资料] 按钮
│   └── 搜索框
├── 筛选栏
│   ├── 来源筛选：[全部] [手动] [思源] [URL] [上传]
│   ├── 分类筛选：[全部] [文档] [笔记] [代码] [链接]
│   └── 标签云
├── 内容区
│   ├── 空状态：引导添加
│   └── 卡片网格 / 列表
└── 右侧面板（点击卡片展开）
    ├── 标题（可编辑）
    ├── 来源标识（思源图标 / 手动图标）
    ├── 标签（可添加/删除）
    ├── 正文（Markdown 编辑器）
    ├── 原始来源链接（URL 时显示）
    └── [保存] [删除] [在思源中打开]
```

### 添加资料流程

```
点击 [+ 添加资料]
├── 选择添加方式
│   ├── [手动输入] → 打开 Markdown 编辑器
│   ├── [输入 URL] → 输入框 + [提取] 按钮 → OpenCLI 提取 → 预览 → 确认添加
│   ├── [上传文件] → 文件选择 → 解析 → 预览 → 确认添加
│   └── [从思源导入] → 选择思源笔记本/文档 → 确认导入
└── 填写元信息
    ├── 标题
    ├── 分类
    ├── 标签（逗号分隔）
    └── [确认添加] [取消]
```

### 与任务流的结合（Wave 3 后续迭代）

1. 任务页输入框上方增加"引用知识库"选择器
2. 系统自动判断调用知识库（后端实现）
3. 知识库页面内直接提问（RAG 模式）
