# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 当前前端界面信息过载、视觉层级混乱、布局碎片化，用户难以快速定位重点。
  2. TopBar 同时承载品牌、导航、6+ 个状态指标和按钮，严重拥挤。
  3. Sidebar 垂直堆砌 5-6 个同质卡片，信息密度过高但无重点。
  4. 空状态大量占位文本占据空间，界面始终显得拥挤。
  5. 底部输入框白色背景与深色主题不协调，视觉断裂感强。
- 对应阶段目标：
  1. 阶段 H（产品差异化与透明执行）：将前端改造为清晰、简洁、现代化的 AI agent 工作台。

## 目标

- 本次要完成什么：
  1. 重构布局架构：左侧图标导航 + 极简顶部 + 单焦点主内容区 + 可选右侧面板。
  2. 建立新的设计 token 体系：收敛背景/文字/状态色，减少视觉噪音。
  3. 简化各视图信息架构：首页极简、任务页单焦点、记录页精简筛选、设置页清晰分组。
  4. 输出完整 design.md / tasks.md / status.md / verify.md。
  5. 按 Wave 渐进实施，每轮有截图验证和回退方案。

## 非目标

- 本次明确不做什么：
  1. 不改后端接口 contract、事件协议或 runtime 状态机。
  2. 不引入大型 UI 组件库（保持 React + Vite + Zustand 轻栈）。
  3. 不一次性重写全部业务逻辑，只改布局和视觉层。
  4. 不切主推进项，本 change 作为前端重设计的独立工作区。

## 验收口径

- 通过标准：
  1. proposal/design/tasks/status/verify 五件套齐备。
  2. 新布局在 4 个核心视图（home/task/logs/settings）均已落地。
  3. 各视图截图通过设计一致性检查（留白、层级、对齐）。
  4. `npm run build` 通过，无新增 TypeScript 错误。
  5. 用户主观验收：界面"不乱了"。
