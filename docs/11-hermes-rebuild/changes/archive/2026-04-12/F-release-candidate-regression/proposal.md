# 变更提案

## 背景

- `F-03` 已完成发布与回滚文档收口，下一步是 `F-04` 发布候选回归与故障注入。
- 当前缺少统一聚合脚本来一次性验证候选包在“回归 + 故障注入”下的稳定性。
- 用户要求继续后端主线推进，本轮不改前端代码。

## 目标

- 新增发布候选聚合验收脚本，统一执行回归链与故障注入链。
- 产出 `tmp/stage-f-rc/latest.json`，形成 F-04 可复核证据。

## 非目标

- 本轮不做 Windows 新机 10 分钟实机验证（归属 `F-05`）。
- 本轮不进行 Gate-F 最终评审（归属 `F-G1`）。
- 本轮不触碰前端实现细节。

## 验收口径

- `run-stage-f-rc-acceptance.ps1` 可执行并输出 `status=passed`。
- 报告中回归链与故障注入链通过率达到 100%（本轮 3 轮样本）。
- 文档索引与总表状态同步到 `F-04` 已完成。
