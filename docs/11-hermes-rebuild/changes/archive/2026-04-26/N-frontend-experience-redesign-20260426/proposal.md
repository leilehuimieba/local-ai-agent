# Proposal: 前端体验重设计

## 背景

当前前端在三个核心场景存在体验缺口：

1. **聊天/任务页布局反直觉**：恢复结论、当前动作等状态卡片堆在消息流上方，用户提问反而在底部，阅读顺序与对话时间线相反。
2. **工作历史页左侧空洞**：记录页仅有 62px 图标 Rail，右侧内容区拥挤，左侧大面积浪费且无导航辅助。
3. **知识库形态落后**：当前知识库仅为列表展示，缺乏 NotebookLM 式的"资料源 + 基于资料对话"能力，也缺乏 Obsidian 式的图谱可视化与节点漫游能力。

## 目标

按 P0→P1→P2 逐步推进：

1. **P0 — 聊天对话流改造**：把任务页从"状态面板堆叠"改成"用户-AI 交替对话流"，AI 的思考过程、工具调用、结论按时间顺序自然展开，参考 Codex 的流式执行呈现方式。
2. **P1 — 工作历史页左侧导航面板**：为记录页补充时间范围、状态、标签等筛选面板，与任务页左侧面板保持体验一致性。
3. **P2 — 知识库 NotebookLM + Obsidian 化**：
   - 资料管理：Sources 式资料源上传与管理
   - 对话学习：基于选中资料的 AI 问答
   - 图谱可视化：Obsidian Graph View 风格的节点-连线图谱，点击节点进入笔记

## 非目标

1. 不改后端 API 协议（前端适配现有 API，如需新接口另提变更）。
2. 不改 Rust runtime 执行逻辑。
3. 本次不实现真正的本地文件系统图数据库存储（图谱可用内存数据 + localStorage 模拟）。
4. 不重构全局状态管理（继续用现有 zustand/store 模式）。

## 验收标准

1. 任务页消息按时间顺序排列，最新消息在底部，用户滚动可查看完整对话流。
2. AI 执行过程（思考、工具调用、结果）以可折叠/展开的卡片形式嵌入对话流，不阻断阅读。
3. 记录页左侧具备筛选面板，且与任务页展开/收起行为一致。
4. 知识库具备 Sources 列表、基于资料对话、Graph View 三个可切换视图。
5. 所有改动通过前端测试、TS 编译、全量回归。
