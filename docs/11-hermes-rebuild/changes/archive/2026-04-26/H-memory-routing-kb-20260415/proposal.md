# H-memory-routing-kb-20260415（proposal）

更新时间：2026-04-16
状态：草案

## 背景

1. `H-learning-mode-browser-20260415` 已完成学习模式最小闭环，但当前记忆路由仍停留在“文章级写入 + 注入预览”的最小能力。
2. 阶段 H 路线要求继续补齐 H-05：明确“记什么、何时写、何时读、何时注入”，降低记忆污染与误注入风险。
3. 当前项目已具备 `POST /api/v1/learning/memory/write`、长期记忆存储与注入预览基础，可在此基础上继续收紧策略。

## 目标（一句话）

建立学习模式下可解释、可回退、可审计的记忆路由与知识沉淀最小治理闭环。

## 范围

### In-Scope

1. 学习文章记忆分类与写入判据收紧。
2. 注入预览、命中解释与预算约束补齐。
3. 误注入/回退演练与证据口径落盘。
4. H-05 change 工作区与提审材料。

### Out-of-Scope

1. 语义检索大改或向量库接入。
2. 前端/插件完整联调。
3. 多站点记忆治理统一抽象。
4. 运行时主循环重构。

## 验收标准

1. 记忆命中有效率 >= 80%。
2. 误注入率 <= 5%。
3. 记忆污染可检测、可回退，回退可用率 = 100%。
4. 聚合证据与提审包可复跑、可审计。

## 风险与回退

1. 风险：学习文章摘要过长或过泛，导致注入噪声上升。
2. 风险：写入阈值过宽，导致长期记忆被低价值内容污染。
3. 回退：关闭自动写入，保留手动确认写入与注入预览。
