# Proposal: 产品化封装上线向导

## 背景

项目已经完成竞品用户迁移体验优化，但上线前检查、安装包构建、Doctor 诊断、发布候选验证仍主要以脚本和文档形式存在。用户不应该记脚本名，也不应该直接面对内部验证流程。

## 目标

做一个完整的用户向导，把产品化封装流程翻译成清晰的发布前工作流：

1. 上线前检查
2. 安装包构建
3. Doctor 诊断
4. 发布候选验证
5. 发布建议与风险提示

## 非目标

1. 不重写现有脚本。
2. 不引入真实打包系统之外的新发布平台。
3. 不改变 runtime/gateway API 协议。
4. 不执行自动发布或推送。

## 验收标准

1. 前端有明确“上线向导”入口。
2. 向导能用用户语言展示步骤、脚本、产物、通过标准和失败处理。
3. 首页恢复/首次入口能引导用户进入产品化封装。
4. 前端测试、TS 编译和全量回归通过。
