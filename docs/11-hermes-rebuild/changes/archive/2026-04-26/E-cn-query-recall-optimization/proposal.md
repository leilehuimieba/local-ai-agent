# 变更提案

## 背景

- 本次变更要解决的问题：
  1. `E-knowledge-base-activation` 的 `T25` 评测中，固定样例存在 `fallback query` 依赖。
  2. 中文主查询在外部 recall 空结果时，缺少统一的“语义锚点补全”策略。
  3. 需要在不改 schema 的前提下提升中文 query 的一次命中能力。
- 对应阶段目标：
  1. 阶段 E / Gate-E 下的知识检索稳定性优化。

## 目标

- 本次要完成什么：
  1. 为外部 recall 增加中文查询的自动 fallback 生成逻辑（基于关键词锚点）。
  2. 保持本地优先主链路不变，仅在外部结果为空时触发补充召回。
  3. 补齐单测与验证记录，形成可追溯优化证据。

## 非目标

- 本次明确不做什么：
  1. 不调整主数据库 schema。
  2. 不引入新的外部服务或模型依赖。
  3. 不改 Gate-E 阈值口径。

## 验收口径

- 通过标准：
  1. 中文 query 命中空结果时可自动生成 fallback 查询锚点。
  2. 英文 query 不触发该 fallback 逻辑。
  3. `knowledge::tests` 全量通过，新增用例覆盖中文 fallback 生成。
