# 变更提案

## 背景

- 当前执行文档存在“阶段/Gate/活跃 change”多源记录，已出现口径冲突。
- 项目推进和验收依赖状态字段一致性，状态漂移会直接影响任务选择和 Gate 判定。
- 本轮属于中等以上文档治理任务，需要以独立 change 收口。

## 目标

- 建立单一状态主记录，统一“当前阶段、当前 Gate、当前活跃 change”判定来源。
- 将入口文档与未归档 change 状态头部改为引用主记录，减少并行口径。
- 保持现有执行流程不变，只收口状态源，不扩展实现范围。

## 非目标

- 本轮不改 Runtime、Gateway、Frontend 代码逻辑。
- 本轮不批量改写 `docs/archive/` 历史文档。
- 本轮不重做阶段计划内容和历史 Gate 证据文本。

## 验收口径

- 新增状态主记录文件，且明确“单一事实源”地位。
- `docs/README.md`、`文档阅读与执行指引.md`、`changes/INDEX.md` 完成主记录引用改造。
- 未归档 change 的 `status.md` 头部“当前阶段”改为引用主记录，不再硬编码阶段值。
