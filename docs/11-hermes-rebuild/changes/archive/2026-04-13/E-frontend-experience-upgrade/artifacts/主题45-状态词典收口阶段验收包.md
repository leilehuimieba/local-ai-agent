# 主题 45：状态词典收口阶段验收包

- 更新时间：2026-04-13
- 对应 change：`E-frontend-experience-upgrade`
- 对应任务：Wave 1 主题 45（状态词典收口阶段验收包）

## 1. 验收范围

本次验收针对“状态词典收口”阶段性成果，不新增页面，不改 API 合同，聚焦跨页面状态词汇与状态样式的一致性。

覆盖页面与模块：

1. 任务页（主线程、工具条、检查器、输入区状态反馈）
2. 调查层（摘要区、事件时间线、模式提示）
3. 记录页（时间线、详情栏、治理状态标签）
4. 设置页（模块状态、Provider 状态、反馈态与总览状态）

## 2. 统一词典基线

统一状态短词：

1. `等待中`
2. `进行中`
3. `待确认`
4. `完成`
5. `失败`

统一映射主入口：

1. `frontend/src/runtime/state.ts`
2. `frontend/src/chat/chatResultModel.ts`
3. `frontend/src/events/EventTimeline.tsx`
4. `frontend/src/history/components/HistoryTimelineSection.tsx`
5. `frontend/src/settings/SettingsSections.tsx`
6. `frontend/src/settings/ProviderCredentialsSection.tsx`
7. `frontend/src/settings/StatusCard.tsx`
8. `frontend/src/workspace/ContextSidebar.tsx`

## 3. 覆盖页与映射点清单

### 3.1 任务页

1. 主线程状态卡标题与徽标短词统一到词典口径。
2. 工具条 `statusLine` 与 tone 输出基于统一映射，不再出现“文案进行中、颜色空闲”分叉。
3. 检查器状态 class 映射由运行时词典集中处理，同义词兼容不再散落在页面组件。

### 3.2 调查层

1. 摘要标题与建议文案复用统一等待态映射。
2. 事件流状态优先消费失败/待确认/完成信号，减少按事件类型推断导致的别名漂移。

### 3.3 记录页

1. 时间线状态徽标与条目 tone 基于同一状态键输出。
2. 治理状态 class 通过统一词典映射，不再本地硬编码 `status-*`。

### 3.4 设置页

1. 模块状态、Provider 状态、反馈态和总览状态统一进入词典映射链路。
2. `ExternalConnection` 状态从 class 直写切换为状态键映射，语义色与任务页一致。

## 4. 回归结果

### 4.1 构建回归

1. 命令：`frontend/` 目录执行 `npm run build`
2. 结果：通过（`tsc -b && vite build`）

### 4.2 人工核对结果

1. 跨页面状态词汇未再出现“运行中/执行中/任务运行中/已完成”并行口径。
2. 任务页、调查层、记录页、设置页状态徽标语义色保持一致。
3. 历史兼容词（如 `等待任务`、`已归档`）仍可命中预期样式，不影响旧数据回放。

## 5. 已知遗留

1. 记录页筛选维度仍需结构化分组（进入主题 46）。
2. 设置页诊断信息尚未按“当前状态/影响范围/建议动作”重排（进入主题 47）。
3. 前端最小回归脚本尚未形成固定模板（进入主题 48）。

## 6. 下一阶段基线

进入下一波优化前，执行口径以本验收包为基线：

1. 所有新增状态文案必须优先复用 `runtime/state.ts` 统一词典。
2. 新增状态 class 禁止页面内直写 `status-*` 分支，优先用状态键映射。
3. 每刀改动结束必须同步 `tasks.md / status.md / verify.md` 与构建回归。
4. 若出现跨页面词汇或语义色分叉，先修正词典映射再继续扩展 UI 优化。
