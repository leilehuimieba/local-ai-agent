# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 现有敏感信息治理仅覆盖密钥/令牌类标记，未覆盖常见隐私字段模式。
  2. 入库拦截与召回审计各自维护敏感判断，存在规则漂移风险。
  3. backlog 第 3 项要求扩展邮箱、手机号、身份证的模式识别。
- 对应阶段目标：
  1. 阶段 E / Gate-E 的知识治理与审计口径收敛。

## 目标

- 本次要完成什么：
  1. 扩展敏感识别规则，覆盖邮箱、手机号、身份证模式。
  2. 统一入库拦截与召回审计使用同一套敏感判断实现。
  3. 补齐单元测试与验证记录，形成可复核证据。

## 非目标

- 本次明确不做什么：
  1. 不引入 schema 迁移。
  2. 不新增外部依赖服务。
  3. 不调整 Gate-E 其他指标阈值。

## 验收口径

- 通过标准：
  1. `knowledge_store` 对邮箱/手机号/身份证内容触发入库拦截。
  2. `knowledge` 对邮箱/手机号/身份证 query 触发审计 source 脱敏。
  3. 相关测试通过并记录在 `verify.md`。
