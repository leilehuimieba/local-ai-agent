# H-visibility-runtime-20260415（proposal）

更新时间：2026-04-15
状态：草案

## 1. 背景

1. 当前执行链路已有日志与事件，但用户侧仍存在“黑箱感”：
   - 不知道智能体当前在做什么
   - 不知道是否卡住或等待
   - 失败后不知道接下来该做什么
2. 本地智伴核心价值之一是“低成本协作”，透明执行是前置能力。

## 2. 目标（一句话）

建立可视化透明执行主线：让用户实时看到“当前阶段、当前任务、当前工具、进度心跳、失败分流与接管路径”。

## 3. 范围

### 3.1 In-Scope

1. 运行阶段可视化（Analyze/Plan/Act/Verify/Finish）。
2. 当前任务卡（目标、动作、等待原因、超时计时）。
3. 工具执行反馈（摘要预览 + 原文引用路径）。
4. 卡住检测与心跳。
5. 失败分流状态（自动重试/等待确认/人工接管/终止）。

### 3.2 Out-of-Scope

1. 大规模重写前端信息架构。
2. 新增复杂动画系统。
3. 与透明执行无关的功能扩展。

## 4. 成功标准

1. 主链路可视化覆盖率 >= 98%。
2. “无反馈卡住>60秒”占比 <= 1%。
3. 失败事件 100% 带“原因+下一步建议”。

## 5. 风险与回退

1. 风险：状态字段跨端漂移导致展示错误。
2. 风险：心跳频率过高引发噪声。
3. 回退：保留旧展示逻辑开关，新字段异常时回退旧口径。
