# 任务清单

## 执行规则

1. 任务按 `G1 -> G2 -> G3 -> G4 -> G5` 顺序推进，未过当前子门禁不得进入下一个。
2. 同一时刻仅允许一个 in-progress 任务。
3. 每个任务都必须有可验证完成判据和回退动作。
4. 未补 `verify.md` 证据前，不得打勾完成。
