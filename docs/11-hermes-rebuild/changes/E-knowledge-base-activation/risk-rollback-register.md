更新时间：2026-04-13  
所属阶段：阶段 E / Gate-E（执行中）  
适用范围：`E-knowledge-base-activation` 全部子门禁（`G1-G5`）

## 1. 使用规则

1. 每个子门禁至少保留 1 条可执行回退动作，触发即执行，不做口头回退。
2. 回退动作执行后必须补核验记录，并同步 `verify.md`。
3. 回退优先级：先保障主链路可用，再恢复增强能力。
4. 主事实源不变：`SQLite/主数据库` 不参与本轮替换。

## 3. 回退执行顺序

1. 先执行“停止放大风险”的动作：关闭开关、停止外部服务、冻结提审。
2. 再执行“恢复主链路”的动作：回到本地写入/检索单路径。
3. 最后执行“证据补齐”：补 `verify.md` 的触发、动作、结果与残留风险。

## 4. 回退演练要求（供 T28 复用）

1. 至少覆盖 `EK-R03`（ingest 回退）和 `EK-R04`（recall 回退）两条。
2. 每次演练必须记录：
   - 触发条件
   - 实际执行命令或步骤
   - 回退后验证结果
   - 是否恢复到主链路稳定状态
