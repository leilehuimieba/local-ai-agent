# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 当前知识库基础能力已接入，但运行态 `knowledge_count` 偏低，知识层沉淀不足。
  2. 主链路任务大量走 `agent_resolve`，现有知识写入来源未覆盖该路径，导致“有结果但不入库”。
  3. 部分有效结果摘要较短时会被拦截，知识写入命中率偏低。
  4. 记忆与知识工具选型已完成初步调研，需进入可执行规划，避免后续实现偏离主线。
- 对应阶段目标：阶段 E（交互与网关统一）下的“可观测与可复盘一致性”补齐，确保知识层在真实任务中可持续积累。

## 目标

- 本次要完成什么：
  1. 扩展知识写入来源，覆盖 `agent_resolve` 成功结果。
  2. 收口知识摘要策略，短摘要场景回退到最终答案摘要，减少误拦截。
  3. 在不替换主数据库的前提下，增加可插拔外部记忆服务接入路径（优先 `cortex` 本地化适配）。
  4. 打通知识消费链路（Markdown 导出 + Obsidian 图谱可视化）并建立固定验收样例。
  5. 将全量执行任务拆分为最小 task，建立门禁与回退规则，保证可持续推进。

## 非目标

- 本次明确不做什么：
  1. 不重做知识库 schema 或存储结构。
  2. 不把思源或 Obsidian 作为主事实源，二者仅作为可插拔消费/沉淀层。
  3. 不在本刀引入重型语义检索重构或全链路迁移。
  4. 不在未通过 Gate 子门禁前提前宣告完成。

## 验收口径

- 通过标准：
  1. `agent_resolve` 成功结果可进入知识类型判定链路。
  2. 短摘要不会因 `<20` 直接被误拦截（可回退到 `final_answer` 摘要）。
  3. 新增单测通过，且知识库过滤规则仍能拦截失败/低价值结果。
  4. 真实业务样例“学习4级备考”可完成“写入 -> 检索 -> 图谱查看”的端到端链路。
  5. 固定评测集 Top-5 命中率达到 `>=70%`，并可定位失败原因。
  6. 安全基线生效：外部服务启用访问令牌，默认不暴露未鉴权 API。
