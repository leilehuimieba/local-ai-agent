# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 当前知识库基础能力已接入，但运行态 `knowledge_count` 偏低，知识层沉淀不足。
  2. 主链路任务大量走 `agent_resolve`，现有知识写入来源未覆盖该路径，导致“有结果但不入库”。
  3. 部分有效结果摘要较短时会被拦截，知识写入命中率偏低。
- 对应阶段目标：阶段 E（交互与网关统一）下的“可观测与可复盘一致性”补齐，确保知识层在真实任务中可持续积累。

## 目标

- 本次要完成什么：
  1. 扩展知识写入来源，覆盖 `agent_resolve` 成功结果。
  2. 收口知识摘要策略，短摘要场景回退到最终答案摘要，减少误拦截。
  3. 补齐单测与验证证据，确保放量不引入失败噪声。

## 非目标

- 本次明确不做什么：
  1. 不重做知识库 schema 或存储结构。
  2. 不改前端信息架构，仅补必要验证证据。
  3. 不在本刀引入语义向量检索。

## 验收口径

- 通过标准：
  1. `agent_resolve` 成功结果可进入知识类型判定链路。
  2. 短摘要不会因 `<20` 直接被误拦截（可回退到 `final_answer` 摘要）。
  3. 新增单测通过，且知识库过滤规则仍能拦截失败/低价值结果。
