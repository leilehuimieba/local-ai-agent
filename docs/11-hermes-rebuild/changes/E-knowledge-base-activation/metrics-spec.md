# E-knowledge-base-activation 指标口径（T03）

更新时间：2026-04-13  
所属阶段：阶段 E / Gate-E（执行中）  
适用范围：`E-knowledge-base-activation` 的 `G1-G5` 全流程验证

## 1. 口径目标

1. 统一“写入成功率、召回命中率、延迟、污染率”定义，避免多处口径漂移。
2. 指标定义只基于可落地数据源，不依赖手工主观判断。
3. 后续任务（`T09/T10/T25/T26/T28`）必须引用本文件口径。

## 2. 指标定义

| 指标ID | 指标名称 | 定义 | 计算公式 | 统计窗口 | 数据源 | 目标阈值 |
|---|---|---|---|---|---|---|
| M01 | 写入成功率 | 知识写入请求中成功写入的比例 | `successful_ingest / total_ingest * 100%` | 单轮任务 + 日汇总 | 运行日志、adapter 响应 | `>=95%`（冒烟），`>=98%`（稳定） |
| M02 | 召回 Top-5 命中率 | 固定评测集中，Top-5 结果命中预期要点的比例 | `hit_cases / total_cases * 100%` | 固定评测集批次 | `fixtures/cet4-acceptance-cases.jsonl` + recall 输出 | `>=70%` |
| M03 | 召回 P95 延迟 | recall 请求耗时的 P95 | `p95(recall_latency_ms)` | 每 50 次 recall 滚动窗口 | 运行日志 `duration_ms/tool_elapsed_ms` | `<=1500ms`（本地优先） |
| M04 | 污染率 | 被判定为低价值/错误/重复的知识条目占比 | `polluted_records / new_records * 100%` | 日汇总 | 清洗规则报告、抽样复核 | `<=10%` |
| M05 | 回退可用率 | 触发回退后主链路恢复成功比例 | `rollback_success / rollback_attempts * 100%` | 回退演练批次 | 回退演练记录（`T28`） | `=100%`（演练阶段） |

## 3. 命中判定规则（用于 M02）

1. 每条样例先定义 2-3 个“预期要点”（见 `expected_checks`）。
2. Top-5 结果中至少 1 条同时命中“主题 + 动作”两个维度，计为命中。
3. 对 `write/read` 类样例，必须同时满足：
   - 结果命中
   - 工具执行成功且 artifact 可打开

## 4. 统计与报告格式

1. 单次执行报告最少字段：
   - `run_id`
   - `metric_id`
   - `value`
   - `window`
   - `source_path`
   - `passed`
2. 日汇总最少字段：
   - `date`
   - `M01-M05` 当日值
   - 失败样例计数
   - 主要失败原因 Top-3

## 5. 异常与口径变更规则

1. 指标未达标时，先记录失败原因，不允许直接改阈值。
2. 如确需调整阈值，必须同时更新：
   - `tasks.md` 对应任务完成判据
   - `verify.md` 的指标解释
3. 口径变更需注明生效日期，默认不追溯改写历史结果。
