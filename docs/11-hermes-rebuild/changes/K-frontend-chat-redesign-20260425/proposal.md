# Proposal: 前端聊天体验重新设计

## 背景

当前前端界面信息过载，对非技术用户不友好：
- 任务页顶部有"运行/会话/工作区"等复杂元数据（TaskPageToolbar）
- 底部有"调查与执行轨迹"面板，展示大量事件细节
- Composer 是 fixed 定位，视觉上偏右
- AI 回复使用固定模板（执行结论/结果说明/验证结果/建议下一步）
- 日志 drawer 塞了太多筛选和分类，新用户无法消化

## 目标

把产品从"AI 执行终端工作站"改造成"像 ChatGPT 一样好上手的智能助手"。

## 核心原则

1. **小白优先**：第一次打开就能发消息，不需要理解任何技术概念
2. **聊天为核心**：所有交互围绕对话流展开
3. **渐进式披露**：复杂信息默认隐藏，需要时再展开
4. **视觉简洁**：减少边框、卡片、标签，多用留白

## 范围

### 任务页（主工作区）
- 删除 TaskPageToolbar（运行/会话/工作区/展开调查层/查看记录页）
- 删除底部调查面板（事件时间线）
- 删除"任务主线程"头部和状态标签
- 消息流改为标准聊天布局：用户右上，AI 左下
- Composer 改为流式定位（跟随内容，像 ChatGPT）
- 空状态时 Composer 在页面中央
- AI 回复纯文本优先，只在有工具调用时显示过程

### 记录页
- 从 drawer 改回独立全页
- 按时间线展示，默认只显示结果摘要
- 点击摘要展开详情（工具调用、验证结果等）
- 隐藏复杂的筛选和分类（全部类型/全部审计/仅确认等）

### 全局
- 左侧 Rail 保留（简化图标）
- 页面切换动画保留
