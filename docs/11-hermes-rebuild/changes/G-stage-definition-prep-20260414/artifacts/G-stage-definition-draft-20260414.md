2. 建立证据保鲜机制，避免 Gate 报告长期失效。
3. 建立变更准入与回归策略，控制回归风险。

## 3. 必做任务（草案）

1. `G-01` 证据保鲜策略与自动复跑入口（Gate 证据时效治理）。
2. `G-02` 发布后巡检与告警审计流程固化（含 warning 跟踪闭环）。
3. `G-03` 回归基线管理（最小回归集 + 失败分流）。
4. `G-04` 运行手册与值守职责收口（可执行 runbook）。

## 5. Gate-G（草案）

1. 关键验收脚本具备自动复跑入口，且证据时效不超过约定阈值。
2. warning 告警项具备责任人、追踪编号、到期时间三要素。
3. 最小回归基线执行通过率达到既定阈值（建议 >=95%）。
4. 无未处理 P0/P1 阻塞项。

## 6. 失败回退（草案）

1. 若 Gate-G 指标不达标，冻结新特性并回退到最近稳定发布基线。
2. 恢复期只允许修复类变更，禁止扩项。
3. 所有回退动作必须产出审计证据并补复盘记录。
