# 变更提案

## 背景

- 当前 `阶段计划总表` 已定义 A~F 阶段目标与 Gate，但“单轮对话可直接领取并完成”的最小任务颗粒度不足。
- 用户明确要求建立“全路线最小任务分解”，确保后续每次对话都能按任务文档推进并沉淀证据。
- 对应阶段目标：阶段 C 收口后的执行治理增强，降低跨阶段推进的不确定性与口径漂移。

## 目标

- 产出一份覆盖 A~F 的最小任务分解总表，包含任务 ID、前置依赖、单轮动作、DoD、验证方式、证据落点、状态。
- 给出“每次对话执行协议”，把任务领取、实现、验证、文档回写固化为标准流程。
- 将该总表接入现有执行入口，避免后续再靠聊天临时拆分任务。

## 非目标

- 本次不直接实现阶段 D/E/F 的功能代码。
- 本次不修改阶段目标与 Gate 阈值。
- 本次不替代各 change 的五件套，只提供跨阶段任务索引与最小执行颗粒度。

## 验收口径

- 新增 `docs/11-hermes-rebuild/stage-plans/全路线最小任务分解总表.md`。
- 总表覆盖 A~F，且每个阶段至少包含可执行最小任务与 Gate 任务。
- `阶段计划总表` 与 `changes/INDEX.md` 完成入口挂接，便于按文档推进。
