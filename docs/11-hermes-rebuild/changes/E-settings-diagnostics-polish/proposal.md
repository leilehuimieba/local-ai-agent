# 变更提案

## 背景

- 本次变更要解决的问题：
  1. 设置页“打开诊断信息”依赖浏览器弹窗，容易被拦截，导致操作反馈不稳定。
  2. 诊断时间在同一页出现原始 ISO 与本地可读格式混用，可读性不一致。
  3. 记录页搜索在错误场景关键词命中不稳定，需要补齐检索字段与状态词。
- 对应阶段目标：阶段 E（交互与网关统一）下的前端交互一致性与可读性收口。

## 目标

- 本次要完成什么：
  1. 将“打开诊断信息”改为稳定可用的导出行为，避免弹窗依赖。
  2. 统一诊断检测时间文案为本地可读格式。
  3. 收口记录页搜索关键词覆盖与重复 key 告警，提升回归稳定性。

## 非目标

- 本次明确不做什么：
  1. 不改后端诊断接口结构。
  2. 不重做设置页整体布局与样式体系。
  3. 不处理所有表单可访问性告警，仅处理当前主题直接相关问题。

## 验收口径

- 通过标准：
  1. 设置页诊断动作不再出现“浏览器阻止弹窗”导致的失败提示。
  2. 诊断时间在设置页统一展示为同一口径。
  3. 记录页搜索可稳定命中 `runtime_unavailable` / `运行时不可达` / `错误`。
  4. 前端构建通过，真人点击回归通过。
