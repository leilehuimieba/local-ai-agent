# H-learning-mode-browser-20260415（proposal）

更新时间：2026-04-15
状态：草案

## 背景

1. 本地智伴核心差异之一是“降低学习成本 + 可持续成长”。
2. 当前项目缺少学习模式下的页面理解与个性化辅助闭环。
3. 参考 yilan 可快速获得页面抽取与侧栏交互基线，但需按本地智伴架构重路由。

## 目标（一句话）

建立学习模式浏览器辅助能力：在用户浏览页面时提供解释、翻译、价值判断与个性化学习建议，并把有效结论沉淀为可复用记忆资产。

## 范围

### In-Scope

1. 学习模式开关与请求协议。
2. 页面结构化抽取（标题/段落/选区/术语）。
3. 解释与翻译卡片。
4. 价值判断与“该关注什么/该学什么”建议。
5. 与本地记忆路由联动（写入/读取/注入）。

### Out-of-Scope

1. 大规模浏览器插件生态建设。
2. 与学习无关的网页自动化功能扩展。
3. 直接在插件端做长期记忆决策。

## 验收标准

1. 页面解析成功率 >= 95%（样本集）。
2. 解释/翻译准确率 >= 90%（人工评测）。
3. 个性化建议相关性 >= 85%。
4. 学习结论沉淀可回放率 = 100%。

## 风险与回退

1. 风险：页面噪声导致建议失真。
2. 风险：插件端与本地网关职责混淆。
3. 回退：关闭学习模式或降级为“仅解释/翻译，不沉淀”。
