# H-remediation-playbook-20260415（proposal）

更新时间：2026-04-16
状态：已冻结（工作区最小口径）

## 背景

1. 阶段 H 路线要求补齐 H-02“低成本修复与接管指引”，当前已切换到对应 change 工作区，但仍缺真实自动修复证据。
2. H-01、H-04、H-05 已分别完成签收或收口，Gate-H 仍缺 H-02 / H-03 输入，无法直接进入阶段签收。
3. 当前项目已有 `doctor`、gateway 审计、学习模式证据与 Windows 本地运行基线，可作为 H-02 最小修复与接管口径的复用基础。

## 目标（一句话）

建立 H-02 当前主推进的最小执行闭环，先冻结“自动修复优先 + 不可代执行时分步接管指引”的边界与样本矩阵，再进入最小实现。

## 范围

### In-Scope

1. 明确 H-02 的目标样本、输入边界、验收阈值与证据路径。
2. 冻结最小修复链路：诊断 -> 分类 -> 修复/接管 -> 结果校验 -> 回放。
3. 定义最小证据目录：`tmp/stage-h-remediation/`。
4. 为后续实现准备 H-02 的 proposal/design/tasks/status/verify/review 文档闭环。

### Out-of-Scope

1. 不在本轮新增大规模自动修复实现。
2. 不改造 runtime 主循环或前端完整接管界面。
3. 不扩展到 H-03 MCP/Skills 质量体系。
4. 不提前宣称 H-02 已完成或 Gate-H 可签收。

## 验收标准

1. H-02 文档工作区齐备，可作为当前主推进项的最小执行入口。
2. H-02 的主链路、样本矩阵、证据路径、回退方式和阶段映射已明确写入文档。
3. `current-state.md` 与 `changes/INDEX.md` 已同步切换到 H-02 主推进口径。
4. 最小样本范围、接管指引模板与 `tmp/stage-h-remediation/` 证据结构已落盘。

## 风险与回退

1. 风险：未先冻结修复边界就进入实现，容易扩展成无边界“自动运维”。
2. 风险：接管指引若无样本矩阵和验收口径，后续实现难以评审。
3. 回退：若 H-02 设计评审未通过，回退为“保留工作区草案，不切换为实现态”。
