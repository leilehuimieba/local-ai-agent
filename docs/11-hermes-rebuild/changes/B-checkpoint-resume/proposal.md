# 变更提案

## 背景

- 阶段 B 需要引入统一 run snapshot 与 checkpoint，并补齐恢复机制。
- 本变更直接服务 Gate-B 中的“中断恢复成功率 >= 95%”。

## 目标

- 明确 checkpoint 与 resume 的边界。
- 为运行时恢复链路建立可实现、可验证的最小方案。
- 约束相关实现只围绕阶段 B 的恢复主线推进。

## 非目标

- 本轮不做跨机器恢复。
- 本轮不引入复杂分布式调度。
- 本轮不扩展到阶段 C 以后的权限或审计能力。

## 验收口径

- 能给出 checkpoint 数据边界与恢复入口。
- 能把任务拆成可验证的实现项。
- 能在 `verify.md` 中挂接 Gate-B 所需的验证证据。
