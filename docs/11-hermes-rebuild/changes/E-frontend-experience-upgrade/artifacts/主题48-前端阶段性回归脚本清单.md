# 主题 48：前端重构阶段性回归脚本清单

- 更新时间：2026-04-13
- 对应 change：`E-frontend-experience-upgrade`
- 目标：形成固定“自动脚本 + 人工检查”回归入口，后续每刀复用，不再临时拼接验证项。

## 1. 自动脚本入口

1. 脚本路径：`scripts/run-frontend-wave1-regression.ps1`
2. 默认输出目录：`tmp/frontend-wave1-regression/`
3. 默认产物：
   - `tmp/frontend-wave1-regression/<timestamp>/frontend-build.log`
   - `tmp/frontend-wave1-regression/<timestamp>/summary.json`
   - `tmp/frontend-wave1-regression/latest.json`

执行命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/run-frontend-wave1-regression.ps1
```

可选参数：

1. `-SkipBuild`：跳过自动构建，仅更新回归清单模板。
2. `-OutputDir <path>`：覆盖默认输出目录。

## 2. 固定人工检查项

每次前端改动后，除自动构建外，按以下顺序人工检查：

1. `task_page_status_loop`
   - 任务页提交后等待态、运行态、待确认、完成/失败闭环可读。
2. `investigation_panel_follow_mode`
   - 调查层自动跟随/手动查看切换可感知，回到最新入口可用。
3. `history_filter_and_detail_linkage`
   - 记录页筛选与详情联动正常，状态词汇语义一致。
4. `settings_diagnostics_group_readability`
   - 设置页诊断分组（当前状态/影响范围/建议动作）可快速扫读，状态徽标语义正确。
5. `mobile_layout_check`
   - 手机窄屏下任务页、记录页、设置页关键操作可点击且无布局挤压。

## 3. 结果判定

1. `通过`
   - 自动脚本构建通过，且人工检查 5 项全部通过。
2. `有条件通过`
   - 构建通过，最多 1 项轻微视觉问题，不影响主链路操作。
3. `不通过`
   - 构建失败，或主链路闭环中断，或 2 项及以上检查不通过。

## 4. 与现有验证口径关系

1. 本清单用于主题 38+ 持续小步收口阶段，作为“每刀必跑”的最小回归入口。
2. 更大范围发布回归仍按既有 V1/阶段门禁流程执行，不由本清单替代。
