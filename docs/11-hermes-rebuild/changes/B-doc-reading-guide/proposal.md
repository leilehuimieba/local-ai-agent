# 变更提案

## 背景

- 当前 `docs/` 文档量较大，阅读入口不够聚焦，容易把“历史文档”误当“当前执行口径”。
- 对应阶段目标：阶段 B 已完成主线收口后，需要降低后续协作中的文档查找与口径漂移成本。

## 目标

- 产出一份可直接执行的“文档阅读与口径判断指引”。
- 明确“当前该看什么、哪些默认视为历史参考、现在该推进什么”。
- 将指引挂接到 `docs/README.md` 和 `changes/INDEX.md`，形成稳定入口。

## 非目标

- 不改动运行时代码与脚本逻辑。
- 不对历史文档做批量迁移或重命名。
- 不直接宣告进入下一阶段开发。

## 验收口径

- 通过标准：
  - `docs/11-hermes-rebuild/` 下新增指引文档，内容包含阅读顺序、过时判断、当前状态与下一步建议。
  - `docs/README.md` 增加该指引入口。
  - `docs/11-hermes-rebuild/changes/INDEX.md` 记录本次 change 状态。
