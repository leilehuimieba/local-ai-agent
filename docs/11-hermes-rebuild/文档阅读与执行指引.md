## 4. 一页式阅读顺序（避免看花）

1. `README`：确认当前唯一入口。
2. 总路线：确认阶段目标与 Gate。
3. 阶段计划：确认本阶段必做与失败回退。
4. `changes/INDEX`：确认当前活跃 change。
5. 活跃 change 的 `tasks + status + verify`：确认今天具体动作。
6. 需要细节时，再跳到领域文档或历史证据。

## 5. 如何判断一份文档是否“过时”

满足任一条，可判定为“非当前执行输入”：

1. 位于 `docs/archive/`。
2. 文档自身声明“历史归档、只读参考、不再作为当前执行入口”。
3. 与 `docs/README.md` 的“当前唯一执行入口”不一致。
4. 与 `11-hermes-rebuild` 的总路线或阶段计划冲突。
5. 无法映射到当前阶段和当前 Gate。
