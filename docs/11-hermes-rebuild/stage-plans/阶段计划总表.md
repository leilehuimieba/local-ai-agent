1. 本文是改造执行层唯一阶段计划入口。
2. 所有任务按阶段推进，未过 Gate 不进入下一阶段。

## 2. 阶段 A：基线收口与架构冻结

### 2.1 周期

1. 计划时长：1 周。

### 2.2 必做任务

1. 输出 Hermes 对标矩阵（功能、稳定性、交互、运维、Windows 体验）。
2. 输出现有模块映射表（crate -> 能力 -> 缺口 -> owner）。
3. 输出风险登记册（风险项、触发条件、缓解动作、回退动作）。
4. 冻结 Gate 规则与验收阈值。
5. 冻结 `Runtime Run Contract v1`。
6. 冻结 `Tool Contract v1`。
7. 冻结 `Risk Policy v1`。
8. 冻结 `Windows 发布链路清单 v1`。

### 2.4 Gate-A

1. 所有核心能力均有明确负责模块。
2. 所有阶段均有明确验收指标。
3. 关键风险均有回退路径。
4. 四份 v1 契约文档已冻结并可供 B 阶段直接使用。

### 2.5 失败回退

1. 若对标矩阵分歧无法收敛，冻结新开发，只做事实盘点并提交评审。

1. 50 轮任务无致命崩溃。
2. 中断恢复成功率 >= 95%。
3. 关键事件链路完整可追溯。

### 3.5 失败回退

1. 若恢复链路不稳定，回退到上个稳定事件合同版本，保留检查点格式兼容。

## 4. 阶段 C：工具与权限治理

### 4.1 周期

1. 计划时长：2 周。

### 4.2 必做任务

1. 工具协议统一（入参、出参、错误码、耗时、trace id）。
2. 命令执行后端抽象（PowerShell/CMD/WSL/bash）。
3. 风险分级与确认链统一。
4. 工具调用审计与回放支持。

### 4.3 交付物

1. Tool Contract 文档与实现。
2. 后端适配层实现与测试。
3. 权限策略配置模板。
4. 审计日志查询接口。

### 4.4 Gate-C

1. 高风险动作拦截准确率 >= 99%。
2. 工具调用失败可定位率 >= 95%。
3. 审计日志字段齐全且可追溯。

### 4.5 失败回退

1. 若后端抽象不稳定，先冻结为 PowerShell 主路径，WSL/bash 降级为可选。

## 5. 阶段 D：记忆与技能体系

### 5.1 周期

1. 计划时长：2 周。

### 5.2 必做任务

1. 三层记忆模型结构固化。
2. 检索路由实现（关键词优先，语义预留）。
3. 技能加载与版本治理。
4. 记忆沉淀治理规则与写回策略。

### 5.3 交付物

1. 记忆模型 schema 与迁移脚本。
2. 检索与写回链路测试。
3. 技能运行隔离机制。
4. 跨会话连续性验收样本。

### 5.4 Gate-D

1. 7 天跨会话任务连续性可复现。
2. 核心技能执行成功率 >= 90%。
3. 记忆污染可检测并可清理。

### 5.5 失败回退

1. 若语义检索不稳定，保留关键词检索主路径并关闭语义策略。

## 6. 阶段 E：交互与网关统一

### 6.1 周期

1. 计划时长：2 周。

### 6.2 必做任务

1. CLI/TUI 交互升级（历史、补全、状态、中断）。
2. 网关统一会话协议，优先接入 1-2 条主入口。
3. 会话上下文跨入口一致性校验。

### 6.3 交付物

1. CLI/TUI 体验升级记录。
2. 网关入口联调报告。
3. 跨入口一致性测试报告。

### 6.4 Gate-E

1. CLI/网关上下文一致性 >= 95%。
2. 中断、恢复、确认行为一致。

### 6.5 失败回退

1. 若多入口不稳定，先冻结单入口并保留统一协议层。

## 7. 阶段 F：Windows 产品化与发布

### 7.1 周期

1. 计划时长：2 周。

### 7.2 必做任务

1. 安装/升级路径收口（至少一条稳定分发）。
2. `doctor` 命令完善（依赖、环境、权限、网络）。
3. 发布清单、迁移清单、回滚预案。
4. 发布候选回归与故障注入。

### 7.3 交付物

1. 安装手册与自动化脚本。
2. 发布候选包与验收记录。
3. 回滚操作文档。
4. Windows 新机验证记录。

### 7.4 Gate-F

1. 新机器 10 分钟内完成安装并跑通首任务。
2. 发布候选通过全部门禁测试。
3. 关键问题无 P0/P1 未关闭项。

### 7.5 失败回退

1. 如发布候选不通过，回退到上个稳定 tag，保留数据库前向兼容迁移。

## 8. 阶段 G：发布后稳定运营与持续治理

### 8.1 周期

1. 计划时长：2 周。

### 8.2 必做任务

1. 建立 Gate 证据保鲜策略与自动复跑入口。
2. 固化发布后巡检与告警审计流程。
3. 建立最小回归基线与失败分流策略。
4. 收口运行手册与值守职责。

### 8.3 交付物

1. `G-证据保鲜策略.md`
2. `G-发布后巡检与告警治理.md`
3. `G-最小回归基线清单.md`
4. `G-运行手册与值守规范.md`
5. `tmp/stage-g-*/latest.json` 证据报告

### 8.4 Gate-G

1. 关键验收脚本具备自动复跑入口，证据时效满足阈值约束。
2. warning 告警项具备责任人、追踪编号、到期时间三要素。
3. 最小回归基线执行通过率 >= 95%。
4. 无未处理 P0/P1 阻塞项。

### 8.5 失败回退

1. Gate-G 指标不达标时，冻结新特性并回退到最近稳定发布基线。
2. 恢复期仅允许修复类变更，禁止扩项。
3. 回退动作必须补齐审计证据与复盘记录。

## 9. 依赖与并行策略

1. A 完成后才能进入 B。
2. C 依赖 B 的事件合同稳定版本。
3. D 可与 E 局部并行，但不得破坏 C 的权限契约。
4. F 依赖 B~E 全部 Gate 通过。
5. G 依赖 F-G1 已签收。

## 10. 执行节奏

1. 每周固定一次 Gate 评审。
2. 每日记录阶段进展与风险变化。
3. 每阶段结束必须补齐验收证据。
