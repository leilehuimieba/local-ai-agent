## 1. 阶段定位

阶段 H 的目标不是补“基础命令执行能力”，而是把本地智伴的核心价值收口为可验收产品能力：

1. 降低用户使用电脑的操作成本（尤其 Windows 场景）。
2. 自动修复优先；不可代执行时提供低门槛分步指引。
3. 执行链路透明可视化，用户随时知道智能体在做什么。
4. 把 MCP 与 Skills 建成稳定执行质量的核心基础设施。
5. 建立学习模式 + 页面理解 + 记忆沉淀闭环，形成“可养成”体验。

## 2. 阶段目标（H）

### 2.1 总目标

将“本地运行的智能体”升级为“低成本解决问题 + 可视化可接管 + 可持续成长”的个人协作系统。

### 2.2 结果标准

1. 透明执行覆盖率 >= 98%（运行主链路均有状态、当前动作、进度心跳与可回放证据）。
2. 自动修复 + 人工接管联合完成率 >= 90%（目标场景集）。
3. 失败后可执行指引命中率 >= 95%（用户按步骤可复现完成）。
4. MCP/Skills 主链路成功率 >= 92%，且失败可定位率 >= 95%。
5. 学习模式下“解释/翻译/价值判断”有效性评测 >= 85%（基线样本）。

## 3. 阶段输入与边界

### 3.1 前置输入

1. Gate-G 已签收（发布后稳定运营基线成立）。
2. 已有运行状态机、工具合同、审计链路、记忆基础能力可复用。
3. 当前主状态与活跃 change 以 `current-state.md` 为唯一裁决。

### 3.2 阶段边界（In-Scope）

1. 执行透明可视化（状态、任务、工具调用、进度、失败分流）。
2. 修复优先与接管指引（自动修复/不可代执行时分步指导）。
3. MCP 接入治理与 Skills 执行治理。
4. 学习模式与浏览器页面理解（解释、翻译、价值判断）。
5. 记忆路由策略（记什么、何时写、何时读、何时注入）。

### 3.3 非目标（Out-of-Scope）

1. 大规模 UI 重做（仅围绕透明执行与学习模式必需改造）。
2. 不受控第三方服务深度耦合（先保持本地可控闭环）。
3. 无验收标准的“感觉更智能”类泛化开发。

## 4. 子阶段拆解与交付

### 4.1 H-01 透明执行可视化主线（高优先）

目标：用户可实时看见“现在在做什么、做到哪一步、是否卡住”。

交付物：

1. 统一运行状态面板协议（Analyze/Plan/Act/Verify/Finish + 子状态）。
2. 当前任务卡（目标、当前工具、最近动作、等待原因、超时计时）。
3. 执行心跳与卡住检测（如 > N 秒无进展给出提示与建议动作）。
4. 透明日志轨道（摘要 + 原文外置 + 一键定位证据路径）。
5. 失败分流可视化（自动重试、等待确认、人工接管、终止）。

验收指标：

1. 主链路事件可视化覆盖率 >= 98%。
2. “无反馈卡住 >60s”占比 <= 1%。
3. 失败链路均可见“原因 + 下一步”。

证据建议：

- `tmp/stage-h-visibility/latest.json`
- `tmp/stage-h-visibility/ui-snapshots/`
- `tmp/stage-h-visibility/stall-detection.json`

### 4.2 H-02 低成本修复与接管指引（高优先）

目标：从“会执行命令”升级为“把问题解决完”。

交付物：

1. 常见环境问题诊断/修复 playbook（依赖冲突、路径、权限、端口等）。
2. 自动修复策略：可安全自动执行项优先落地。
3. 不可代执行时的分步指引生成器（逐步、可复制、带预期结果）。
4. 执行前后校验（修复成功判据与回退提示）。

验收指标：

1. 自动修复成功率 >= 70%（目标样本集）。
2. 人工接管步骤成功率 >= 90%。
3. 指引可执行性评分 >= 4.5/5（内部评测样本）。

证据建议：

- `tmp/stage-h-remediation/latest.json`
- `tmp/stage-h-remediation/manual-guides/*.md`
- `tmp/stage-h-remediation/replay-results.json`

### 4.3 H-03 MCP + Skills 执行质量体系（高优先）

目标：把 MCP 与 Skills 变成执行质量“稳定器”。

交付物：

1. MCP 接入分级与超时/重试/降级规范。
2. Skills 路由规则（触发条件、冲突处理、兜底策略）。
3. 技能执行可观测字段（命中原因、失败原因、回退路径）。
4. 最小评测包（核心任务集 + 回归脚本 + 失败样本）。

验收指标：

1. MCP/Skills 主链路成功率 >= 92%。
2. 失败可定位率 >= 95%。
3. 关键技能回归通过率 >= 95%。

证据建议：

- `tmp/stage-h-mcp-skills/latest.json`
- `tmp/stage-h-mcp-skills/evals/*.json`
- `tmp/stage-h-mcp-skills/fallback-cases.json`

### 4.4 H-04 学习模式与页面侧辅助（中高优先）

目标：学习场景下提供个性化解释、翻译、重点建议与价值判断。

交付物：

1. 学习模式开关与会话上下文模板。
2. 浏览器页面理解接口（标题、段落、关键术语、上下文抽取）。
3. 页面侧解释/翻译/要点建议（结合用户背景与当前学习目标）。
4. 内容价值评估（“为何值得看、应关注什么、下一步学什么”）。

验收指标：

1. 页面解析成功率 >= 95%（样本页面集）。
2. 解释/翻译准确性 >= 90%（人工评测）。
3. 学习建议相关性 >= 85%。

证据建议：

- `tmp/stage-h-learning/latest.json`
- `tmp/stage-h-learning/page-cases/*.json`
- `tmp/stage-h-learning/recommendation-eval.json`

### 4.5 H-05 记忆路由与本地知识沉淀（中高优先）

目标：明确“记什么、何时读、何时注入”，避免记忆污染与加载噪声。

交付物：

1. 记忆分类策略（偏好、环境、任务经验、学习笔记、风险记录）。
2. 写入策略（自动写入、确认写入、禁止写入清单）。
3. 检索/注入策略（触发条件、预算、优先级、过期/降权）。
4. 知识库与数据库分工（结构化事实 vs 语义内容）。

验收指标：

1. 记忆命中有效率 >= 80%（目标任务集）。
2. 误注入率 <= 5%。
3. 记忆污染可检测可清理（100% 可回滚）。

证据建议：

- `tmp/stage-h-memory-routing/latest.json`
- `tmp/stage-h-memory-routing/injection-audit.json`
- `tmp/stage-h-memory-routing/rollback-drill.json`

## 6. 风险与回退

1. 风险：可视化层与执行层字段漂移导致 UI“看见的是旧状态”。
   - 缓解：状态合同版本化；端到端合同回归。
   - 回退：回滚到上一版稳定合同并保留新字段兼容读取。

2. 风险：MCP/Skills 路由复杂度上升导致执行不稳定。
   - 缓解：先白名单核心 skills，逐步放量。
   - 回退：降级到“核心技能 + 本地工具”最小链路。

3. 风险：学习模式引入噪声内容污染记忆。
   - 缓解：学习记忆分区、写入阈值、人工确认写入。
   - 回退：关闭自动写入，仅保留手动确认沉淀。

4. 风险：自动修复误操作导致用户环境受损。
   - 缓解：高风险动作强确认 + 预演 + 快照回退提示。
   - 回退：自动修复降级为“建议命令 + 手动执行”。
