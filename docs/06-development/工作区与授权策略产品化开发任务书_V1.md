## 1. 任务定位

本任务书只解决一件事：

> 把当前“能用”的工作区与授权能力收口成产品级体验。

当前任务不是做：

1. 多用户权限系统
2. 企业级 RBAC
3. 复杂权限面板
4. 全盘自动授权
5. 浏览器或桌面自动化权限系统

当前任务要做的是：

1. 工作区切换稳定
2. 当前工作区显示清晰
3. 新目录首次接触提醒完整可用
4. 已记住目录授权可查看和撤销
5. 风险与授权设置区块满足 V2 文档要求

---

## 2. 当前阶段目标

本阶段唯一目标：

> 让工作区与授权策略从“基础能力已存在”升级为“符合产品文档要求的正式能力”。

做到这一步后，系统应具备：

1. 设置页可稳定管理当前工作区
2. 顶部和首页可轻量显示当前工作区状态
3. 新目录首次接触时有完整提示与记忆策略
4. 已记住授权可查看、撤销、立即生效
5. 风险等级展示开关与目录提醒开关可在设置中控制

---

## 3. 本阶段硬约束

开始实现前必须满足：

1. 工作区仍然是系统上下文对象，不升级为首页一级主导航
2. 工作区详细管理统一放在设置页
3. 首页只做轻量显示，不做重型工作区管理台
4. 授权策略必须和风险确认体系保持一致
5. 不引入复杂权限模型

---

## 4. 本阶段必须完成的 5 个模块

### 4.1 `workspace-switching`

职责：

1. 切换当前工作区
2. 保证前后端状态一致
3. 保证切换后任务使用新工作区

### 4.2 `workspace-display`

职责：

1. 在首页轻量显示当前工作区
2. 在设置页完整显示路径与状态

### 4.3 `directory-approval`

职责：

1. 新目录首次接触提示
2. 允许一次性通过或取消
3. 支持“记住此授权”

### 4.4 `approval-management`

职责：

1. 展示已记住授权
2. 撤销已记住授权
3. 撤销后立即生效

### 4.5 `risk-policy-settings`

职责：

1. 控制目录提醒开关
2. 控制风险等级展示开关
3. 统一设置页风险与授权策略区块

---

## 5. 推荐实现顺序

### Phase A：状态收口

先做：

1. 工作区切换状态链路
2. 设置页当前工作区展示
3. 首页轻量工作区显示

通过标准：

1. 工作区切换可稳定生效
2. 当前工作区在 UI 中清晰可见

### Phase B：授权主链路

再做：

1. 新目录首次接触提醒
2. 一次性继续 / 取消
3. 记住授权

通过标准：

1. 新目录首次接触行为完整可用
2. 用户能明确理解授权影响

### Phase C：授权管理与设置收口

最后做：

1. 已记住授权列表
2. 撤销授权
3. 风险与授权设置开关

通过标准：

1. 设置页能完整管理工作区与授权策略
2. 满足 V2 文档对应区块要求

---

## 6. 文件级落点

### 6.1 Go 控制面

优先改：

1. `gateway/internal/api/router.go`
2. `gateway/internal/api/chat.go`
3. `gateway/internal/state/store.go`
4. `gateway/internal/state/confirmation_store.go`

### 6.2 前端

优先改：

1. `frontend/src/App.tsx`
2. `frontend/src/settings/`
3. `frontend/src/workspace/`
4. `frontend/src/events/`

### 6.3 Rust 运行时

一般不要求大改主逻辑。
只要求：

1. 工作区切换后运行时收到正确 `workspace_id / root_path`
2. 新目录授权策略保持与风险确认链路一致

## 7. 文件级任务清单

### 7.1 `gateway/internal/api/router.go`

任务：

1. 完整返回当前工作区
2. 返回可选工作区列表
3. 返回已授权工作区列表
4. 返回目录提醒与风险展示开关

### 7.2 `gateway/internal/state/store.go`

任务：

1. 管理当前工作区
2. 管理已记住授权 ID
3. 管理目录提醒开关
4. 管理风险等级展示开关

### 7.3 `gateway/internal/api/chat.go`

任务：

1. 在发起运行时透传正确工作区上下文
2. 处理确认与授权策略相关输入

### 7.4 `frontend/src/settings/`

任务：

1. 设置页完整展示工作区设置区块
2. 设置页完整展示风险与授权策略区块
3. 提供切换、撤销、开关操作

### 7.5 `frontend/src/workspace/`

任务：

1. 首页轻量显示当前工作区
2. 不做重型管理台

### 7.6 `frontend/src/App.tsx`

任务：

1. 顶部或首页保持当前工作区轻量可见
2. 工作区切换后刷新相关状态

---

## 8. 功能要求

### 8.1 工作区切换

必须满足：

1. 可切换当前工作区
2. 切换后前后端状态一致
3. 切换后运行时使用新工作区

### 8.2 新目录首次接触提醒

必须满足：

1. 首次接触新目录时提示
2. 可继续或取消
3. 可记住此授权

### 8.3 已记住授权管理

必须满足：

1. 可查看已记住授权
2. 可撤销
3. 撤销后立即生效

### 8.4 风险与授权设置

必须满足：

1. 可控制目录提醒开关
2. 可控制风险等级展示开关

---

## 9. UI 要求

必须符合：

1. 首页只轻量显示工作区
2. 设置页承担完整管理职责
3. 风险与授权设置集中在设置页
4. 不新增独立工作区首页

---

## 10. 验收标准

### 10.1 必须通过

1. 工作区切换稳定可用
2. 首页和设置页都能看到当前工作区
3. 新目录首次接触提醒完整可用
4. 已记住授权可查看与撤销
5. 风险与授权设置区块满足 V2 文档要求

### 10.2 不通过情形

1. 工作区切换只是 UI 假切换
2. 首页仍承担重型工作区管理职责
3. 新目录提醒不稳定或无记忆策略
4. 已记住授权无法撤销
5. 风险与授权设置不能真实生效

---

## 11. 给实现型 AI 的明确要求

1. 先做状态真实生效，再做 UI 打磨
2. 先满足设置页管理闭环，再优化首页展示
3. 不要扩成复杂权限系统
4. 完成后必须补对应验收文档

一句话要求：

> 当前任务不是做权限平台，而是把工作区和授权策略收口成真正可用的产品能力。
