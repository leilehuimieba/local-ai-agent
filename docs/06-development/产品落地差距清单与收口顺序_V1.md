## 4. 优先级定义

本文只使用以下优先级：

1. `P0`
2. `P1`
3. `P2`
4. `P3`

含义：

1. `P0`：不做就无法进入产品落地阶段
2. `P1`：不做会明显影响真实使用体验
3. `P2`：做完前两项后应尽快补齐
4. `P3`：增强项，可后置

## 5. 差距清单

### 5.1 P0：SQLite 主存储收口

目标：

1. 把长期记忆和知识索引从当前 `JSON / JSONL` 基线收口到 `SQLite` 主存储
2. 保留 `JSONL` 只做日志、artifact、审计流水

为什么优先：

1. 当前记忆和知识主链路已经成立
2. 但正式数据库主存储还没落下
3. 不先收口存储，后面记忆治理、知识接入、查询稳定性都会继续发散

必须完成：

1. 长期记忆表结构
2. 知识索引表结构
3. 基础索引字段
4. 读写接口收口
5. 与现有 JSONL 的迁移或兼容策略

当前阻断点：

1. 还没有正式数据库主存储
2. 历史数据治理还主要靠过滤和降权

通过标准：

1. 长期记忆正式落入 `SQLite`
2. 知识索引正式落入 `SQLite`
3. 召回主链路优先查询 `SQLite`
4. 旧 JSONL 只保留流水和兼容层

### 5.2 P0：思源外挂知识库接入

目标：

1. 让思源成为“外挂知识库”和“人工沉淀出口”
2. 不让思源承担运行时高频主存储

为什么优先：

1. 你已经明确要结合思源
2. 当前知识沉淀主链路已经有基础
3. 现在正适合把“SQLite 主索引 + 思源外挂正文库”的边界一次定死

必须完成：

1. 思源接入边界
2. 写入思源的条件
3. 从思源抽摘要回写 `SQLite` 的规则
4. 思源检索入口
5. 禁止写入思源的内容边界

当前阻断点：

1. 思源尚未正式进入知识链路
2. 当前知识库还只有内部沉淀，没有外挂知识层

通过标准：

1. 至少支持检索思源条目
2. 至少支持把高价值知识沉淀到思源指定目录
3. 至少支持从思源正文抽摘要回写 `SQLite`

### 5.3 P1：工作区与授权策略产品化

目标：

1. 把当前“能用”的工作区与授权能力收口成产品级体验
2. 满足需求文档里对工作区和新目录授权策略的要求

为什么优先：

1. 工作区是本地智能体的真实使用核心
2. 授权策略如果没收好，产品体验和安全边界都会不稳

必须完成：

1. 工作区切换稳定
2. 当前工作区展示清晰
3. 已记住目录授权可查看
4. 已记住授权可撤销
5. 新目录首次接触提醒完整可用

当前阻断点：

1. 基础能力已存在
2. 但还没有明确达到产品文档要求的完整收口

通过标准：

1. 工作区切换、显示、授权管理均完成
2. 风险与授权设置区块满足 V2 文档要求

### 5.4 P1：前端产品化收口

目标：

1. 把现有前端从“结构可用”收口到“产品可用”
2. 保持你偏好的简洁风格，不再反复改大方向

为什么优先：

1. 当前页面结构已定
2. 再继续改大框架收益不高
3. 现在更需要把状态反馈、信息层级、错误态和交互细节做稳

必须完成：

1. 首页状态层级统一
2. 右侧上下文栏信息密度收口
3. 底部事件流与日志区分更清楚
4. 风险确认卡片更稳定
5. 错误态、空状态、首次使用提示补齐

当前阻断点：

1. 页面已有形态
2. 但还没到“长期使用的产品感”

通过标准：

1. 首页、日志、设置三页体验稳定
2. 过程信息可见但不吵
3. 关键状态清晰，不依赖内部知识才能看懂

### 5.5 P2：回答质量与召回质量收口

目标：

1. 让 `project_answer / knowledge_answer` 更像产品回答
2. 让召回内容更稳定、更聚焦当前项目

为什么放在这里：

1. 这项很重要
2. 但在存储收口和思源接入前，继续细调收益有限

必须完成：

1. README / 架构文档 / 开发任务书优先命中更稳定
2. `07-test` 这类历史验收类内容不再干扰常规项目问答
3. 输出语言更聚焦项目能力与架构，而不是文档导航感

通过标准：

1. 项目问答连续多轮都能稳定引用正确来源
2. 回答不再被历史测试内容主导

### 5.6 P2：产品级总验收

目标：

1. 从“阶段验收”升级到“产品验收”
2. 判断是否能进入真实长期使用阶段

为什么放在最后：

1. 现在做总验收还太早
2. 需要等前面的收口项完成后再做

必须完成：

1. 需求文档逐项验收
2. 前端体验验收
3. 工作区与授权验收
4. 记忆与知识链路验收
5. 多轮真实使用验收

通过标准：

1. 可以正式判定“满足第一版产品落地”

### 5.7 P3：增强项

本阶段不优先做：

1. embedding 检索
2. 复杂记忆评分
3. 知识版本快照系统
4. 多智能体
5. 浏览器自动化
6. 桌面自动化

原因：

1. 这些都不是当前产品落地的最短路径
