## 1. 当前阶段判断

当前项目判断固定为：

1. 首版主链路已成立。
2. 当前版本可作为首版候选版本冻结。
3. 当前总体验收结论为 `通过`。
4. 当前不再回到主链路大补洞阶段。

一句话：

> 现在进入的是“冻结当前可用首版，围绕遗留项治理和下一阶段能力规划推进”的阶段。

---

## 2. 当前冻结范围

当前可视为首版候选版本的冻结范围包括：

1. 单智能体主循环
2. 工具调用、命令执行、文件读写
3. 工作区切换与风险确认
4. 模式切换与模型切换
5. 记忆与知识检索主链路
6. SQLite 主存储
7. 思源外挂知识库接入
8. 首页、日志页、设置页三页产品形态

---

## 3. 当前不作为冻结阻断项的遗留问题

当前遗留问题只按非阻断项处理：

1. `memory_recalled` 尚未独立输出正式事件。
2. 历史数据治理仍可继续扩展到版本化与可解释化。
3. 用户偏好沉淀和失败教训沉淀还需继续做厚。

这些问题会影响下一阶段优先级，不会推翻当前首版候选冻结判断。

---

## 4. 现在不要再做的事

1. 不要回头大改首版页面结构。
2. 不要重新讨论是否采用 Python 作为主后端。
3. 不要把已通过正式验收的能力重新打回“未完成”。
4. 不要再新增和首版无关的一级页面或重型能力域。
5. 不要跳过现有文档口径，直接按旧想法扩写产品。

---

## 5. 下一阶段规划顺序

下一阶段按以下顺序推进：

1. 记忆召回可观测性继续增强，补齐 `memory_recalled` 独立事件。
2. 历史数据治理与召回质量继续收紧，推进版本化与可解释化。
3. 用户偏好沉淀与失败教训沉淀补厚。
4. 第二阶段能力规划，只新增确实服务本地智能体主线的能力。

---

## 6. 第二阶段能力选择原则

后续新增能力必须同时满足：

1. 服务本地单智能体主线。
2. 能复用当前 Go + Rust 主架构。
3. 不明显拉高内存占用和运行复杂度。
4. 能在现有工作区、记忆、知识、日志框架上自然接入。

优先考虑：

1. 记忆质量治理
2. 知识沉淀质量治理
3. 回答质量和召回质量治理
4. 桌面壳层与启动器长期稳定化

后置考虑：

1. 多智能体
2. 浏览器自动化深链路
3. 桌面自动化深链路
4. 语音输入输出
5. 自动 fallback
6. 自动测速
