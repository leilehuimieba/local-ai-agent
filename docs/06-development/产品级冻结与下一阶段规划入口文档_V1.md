## 3. 当前执行顺序（统一治理口径）

当前后续工作统一按下面顺序推进，不再回到“大范围扩能力”：

1. 整理 `V1` 当前唯一执行口径。
2. 做可发布候选，不再扩能力域。
3. 选一个真实场景，补 `3-5` 条任务脚本。
4. 开始一轮真实使用内测。
5. 根据真实使用结果，再开下一阶段任务书。

这些工作会决定下一阶段是否开新任务书，不会推翻当前 `V1` 冻结通过判断。

## 7. 第二阶段能力选择原则

后续新增能力必须同时满足：

1. 服务本地单智能体主线。
2. 能复用当前 Go + Rust 主架构。
3. 不明显拉高内存占用和运行复杂度。
4. 能在现有工作区、记忆、知识、日志框架上自然接入。

优先考虑：

1. 记忆质量治理
2. 知识沉淀质量治理
3. 回答质量和召回质量治理
4. 桌面壳层与启动器长期稳定化

后置考虑：

1. 多智能体
2. 浏览器自动化深链路
3. 桌面自动化深链路
4. 语音输入输出
5. 自动 fallback
6. 自动测速
