---

## 1. 当前前端结构

当前前端固定为 4 个正式页面和 3 个任务页辅助区：

1. 首页
2. 任务页
3. 记录页
4. 设置页

任务页辅助区固定为：

1. 主线程
2. 右侧检查器
3. 底部调查抽屉

禁止重新扩展为概念工作台、多入口控制台或新的一级页面集合。

---

## 2. 页面职责

### 2.1 首页

首页只负责：

1. 根据状态提供快速开始或恢复上下文
2. 显示最近任务摘要
3. 暴露继续最近任务入口
4. 提示当前风险或运行阻塞

首页不负责：

1. 承接主线程对话
2. 展开跨会话调查详情
3. 展示运行时控制项

### 2.2 任务页

任务页是唯一主工作区，负责：

1. 任务输入
2. 结果推进
3. 失败与恢复提示
4. 确认流
5. 当前会话调查

补充规则：

1. 首页提交新任务后，正式执行现场仍然进入任务页
2. 首页不重新承接主线程推进和完整确认处理

新增任务能力优先只接在：

1. 主线程
2. 检查器
3. 调查抽屉

### 2.3 记录页

记录页只负责跨会话调查与复盘，负责：

1. 稳定日志时间线
2. 记录详情阅读
3. 风险、错误、上下文、验证信息复盘

记录页不承接当前会话任务推进。

### 2.4 设置页

设置页只负责运行控制，负责：

1. Runtime Environment
2. Model Providers and Mode
3. Workspace and Approvals
4. Risk and Permissions
5. Memory and Resources
6. Diagnostics and Export

设置页不承接任务结果、记录详情或概览型拼盘。

---

## 3. 新能力优先接入位置

后续新增真实能力时，优先评估以下接入位置：

1. 任务页主线程
2. 任务页检查器
3. 任务页底部调查抽屉
4. 记录页详情区
5. 设置页模块区

如果某项能力不能自然归入以上位置，默认说明边界不清，不新增一级入口。

---

## 3.1 首页状态化补充规则

首页当前正式按状态感知模式工作，只允许以下 3 种状态：

1. `首用态`
2. `恢复态`
3. `阻塞态`

首页状态化规则：

1. 首页是系统首页，不是固定主聊天页。
2. 首用态负责快速开始，恢复态负责继续上下文，阻塞态负责暴露修复入口。
3. 待确认项统一通过首页高优先级横幅提示，不在首页正文展开完整确认卡。
4. 首页输入新任务后，仍然进入任务页作为唯一主工作区。

当前详细口径统一见 [V2首页状态化方案_V1](D:/newwork/本地智能体/docs/10-v2/V2首页状态化方案_V1.md)。

当前实现落点补充：

1. `workspaceViewModel` 负责首页三态派生、最近活动摘要和待确认横幅数据装配。
2. `WorkbenchOverview` 负责首页首用态 / 恢复态 / 阻塞态正文表达。
3. `App.tsx` 负责复用现有任务提交链路，把首页输入和示例任务接入任务页。
4. `TopBar` 只补首页状态语义和高频“新建任务”入口，不改变正式页面边界。
5. 开发态复验首页三态时，可临时使用 `?home_preview=first_use / resume / confirmation / blocked` 做稳定场景预览，不改变正式运行入口语义。

## 4. 已冻结的有限扩展位

当前前端只保留有限、显式、可解释的接入位：

1. `frontend/src/chat/chatResultModel.ts`
   任务页结果表达固定为：
   `summary -> action -> detail -> risk -> next`
2. `frontend/src/workspace/ContextSidebar.tsx`
   检查器信息固定为：
   `task -> action -> repo -> risk`
3. `frontend/src/history/components/HistoryDetailRail.tsx`
   记录详情固定为：
   `basic -> summary -> risk -> metadata -> context -> verification`
4. `frontend/src/settings/SettingsSections.tsx`
   设置模块固定为：
   `runtime -> model -> workspace -> risk -> resources -> diagnostics`

要求：

1. 不建立万能插槽系统。
2. 不提前加未使用 props。
3. 不做配置驱动页面生成器。

---

## 5. 共享组件与页面私有边界

共享 primitives 只保留在：

1. `frontend/src/ui/primitives/SectionHeader.tsx`
2. `frontend/src/ui/primitives/StatusPill.tsx`
3. `frontend/src/ui/primitives/MetricChip.tsx`
4. `frontend/src/ui/primitives/MetaGrid.tsx`
5. `frontend/src/ui/primitives/EmptyStateBlock.tsx`

边界规则：

1. 跨页面稳定复用才允许进入 `ui/primitives`。
2. 页面私有结果表达、详情分组、设置模块编排继续留在页面目录。
3. 不新增大而全通用组件平台。

---

## 6. 样式组织方式

当前样式职责固定为：

1. `frontend/src/styles/tokens.css`
   token 和语义变量
2. `frontend/src/styles/base.css`
   reset、基础行为、焦点、reduced motion
3. `frontend/src/index.css`
   布局、页面、组件、响应式实现
4. `frontend/src/styles/index.css`
   样式入口聚合

不再引入新的样式体系或 CSS-in-JS。

## 7. 新增前端能力接入清单

每次新增前端能力前，先回答：

1. 新能力属于哪个页面或区域。
2. 是否影响任务主线程。
3. 是否增加检查器信息。
4. 是否需要记录页详情承接。
5. 是否需要设置页配置入口。
6. 是否需要新增空态、错态、禁用态。
7. 是否需要补最小回归留证。

如果以上问题回答不清，先不要动代码结构。

---

## 8. 最小增量回归清单

每次新增能力后至少执行：

1. `frontend/` 下执行 `npm run build`
2. 首页继续保持轻量恢复入口
3. 任务页主线程、检查器、调查抽屉未被打坏
4. 记录页详情仍可持续阅读
5. 设置页真实控制项仍可用
6. 手机窄屏未出现明显溢出、挤压、滚动错乱
7. 新能力已补齐空态、错态和必要说明

---

## 9. 不再允许的方向

1. 新增一级页面承接次级能力
2. 重新引入概念工作台、spotlight 拼盘、玻璃感或未来占位噪音
3. 为了可扩展建立大而全插槽系统
4. 为了整齐把页面私有复杂结构强行抽全局
5. 改动 API、状态模型或事件合同语义
