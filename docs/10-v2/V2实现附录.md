## 7. 后续实现注意事项

1. 不要新增顶层 API，优先复用既有 `settings / logs / events / chat` 主链路
2. 不要把 Skills、记忆、模型管理再抬成一级主视图
3. 若继续增强首页，应优先增强“上下文摘要”和“日志可读性”，不要重新把首页堆回大杂烩
4. 如果需要扩展合同，优先补 metadata，而不是扩 endpoint 面
