## 2. 验收范围

本次总体验收只覆盖以下主干能力：

1. Query Engine / Agent 主循环
2. Context Envelope / Prompt 装配
3. Tool Protocol / Tool Registry
4. Permission / Risk Gate
5. Memory / Knowledge Router
6. Artifact / Compaction
7. Verify / Finish 判定
8. Model Adapter / 统一模型接入

## 3. 总体通过条件

只有同时满足以下条件，才可判定主干阶段通过：

1. 主循环存在且稳定。
2. 上下文装配不再散落在零散逻辑中。
3. 工具调用不再是隐式分支，而有统一协议。
4. 高风险动作能进入统一确认路径。
5. 长输出能外置，不再直接压入主上下文。
6. 记忆进入主链路，但仍保持按需读写。
7. Verify 和 Finish 不只是阶段名，而有实际判定逻辑。
8. 模型接入不再散落 provider 细节。
9. 三端构建通过，且关键字段能稳定回流。

## 7. 当前遗留项

### 7.1 非阻断遗留项

当前可接受的非阻断项：

1. 少量结构体字段或 snapshot 类型暂未完全进入消费链路。
2. 部分新结构已建立，但仍偏“主干预留”而非“全链路高频消费”。
3. Rust 侧存在 warning，说明还有清理空间。

### 7.2 阻断项判定标准

如出现以下情况，应直接改判为 `不通过`：

1. 主循环关键阶段未真实接入。
2. Tool Registry 只是声明存在，实际主链路未使用。
3. Verify / Completion 只存在类型定义，没有真实运行逻辑。
4. Model Adapter 没有真正替代散落 provider 调用。
5. Go / Rust / 前端 合同不同步，导致真实链路异常。
