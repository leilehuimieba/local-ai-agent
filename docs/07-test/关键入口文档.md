## 3. 两个高频入口怎么分工

1. [V1回归检查入口_V1](D:/newwork/本地智能体/docs/07-test/V1回归检查入口_V1.md)  
用于执行 `build + 样本 + 文档口径` 的可重复检查，适合改动后快速复跑。

2. [里程碑验收与首版候选汇总_V1](D:/newwork/本地智能体/docs/archive/07-test/里程碑验收与首版候选汇总_V1.md)  
用于回看 M1~M6 里程碑结论、首版候选状态和已并入的历史信息，不作为默认执行入口。

分工原则：

1. 要判断“现在能不能继续往下走”，先看总体验收文档。
2. 要判断“这轮改动有没有把基线打坏”，先跑 V1 回归入口。
3. 要追溯历史阶段结论和已删除文档去向，再看里程碑汇总。
