# 本地智能体 - M4 记忆系统验收记录 V1

更新时间：2026-04-02

状态：已完成实现，待最终人工复核。

## 1. 本轮目标

按 M4 要求，把记忆从散结构收口为结构化 schema，并提供按需 recall digest。

## 2. 实现范围

1. 新增 `MemoryKind`
2. 新增 `StructuredMemoryEntry`
3. 新增 `canonical_kind`
4. 新增 `MemoryDigest`
5. 记忆写入改为先转结构化 schema 再落盘
6. recall 改为生成轻量 digest，供上下文装配使用

## 3. 修改文件清单

1. `crates/runtime-core/src/memory_schema.rs`
2. `crates/runtime-core/src/memory_recall.rs`
3. `crates/runtime-core/src/memory.rs`
4. `crates/runtime-core/src/context_builder.rs`
5. `crates/runtime-core/src/planner.rs`

## 4. 新增结构与字段

### 4.1 MemoryKind

当前支持：

1. `preference`
2. `project_knowledge`
3. `lesson_learned`
4. `daily_note`
5. `task_outcome`

### 4.2 StructuredMemoryEntry

当前字段：

1. `id`
2. `kind`
3. `summary`
4. `content`
5. `scope`
6. `workspace_id`
7. `session_id`
8. `timestamp`

## 5. 运行链路变化

调整前：

1. 记忆直接按自由结构写入
2. recall 主要只用于执行阶段结果拼装

调整后：

1. 写入前统一规范 kind
2. recall 可生成 `MemoryDigest`
3. `RuntimeContextEnvelope` 已接入 `memory_digest`

## 6. 手工测试步骤

1. 发起 `remember:` 或 `写入记忆:` 请求
2. 检查 `data/memory/entries.jsonl`
3. 确认 `kind` 被规范为正式 schema 值
4. 再发相关自然语言请求
5. 检查事件 metadata 中存在 `memory_digest`
