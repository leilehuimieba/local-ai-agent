## 1. 作用

本入口用于把当前 V1 的三类检查收成一个可重复执行入口：

1. 构建检查：`cargo build`、`go build ./...`、`npm run build`
2. 验收样本检查：`tmp/v1-capability-acceptance` 下的本地程序/安装样本
3. 文档口径检查：冻结稿、完成度清单、产品级总体验收文档的关键状态是否一致

## 4. 结果判定

脚本输出的 `overall_status` 只允许以下三种：

1. `passed`
2. `passed_with_warnings`
3. `failed`

判定规则：

1. 只要任一构建或文档口径检查失败，整体为 `failed`
2. 若仅存在 `choco` 这类已记录环境限制样本，整体可为 `passed_with_warnings`
3. 只有全部阻断项通过且无告警时，整体为 `passed`

---

## 5. 建议使用时机

1. 提交首版冻结相关改动前
2. 调整验收样本或文档口径后
3. 进入下一轮能力开发前
4. 宣称“当前状态已稳定”前
