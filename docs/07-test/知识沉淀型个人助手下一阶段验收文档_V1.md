## 12. 两周执行清单验收口径

本节用于和开发任务书第 12 节配套使用。

如果后续实现型 AI 是按“两周执行清单”推进，验收时默认按本节填写，不再另起一套平行口径。

### 12.1 适用范围

本节只验收：

1. `Stage B：知识沉淀闭环`
2. `Stage C：状态理解与下一步建议`
3. `Stage D：最小执行桥接`

本节不验收：

1. 多智能体
2. 浏览器深链自动化
3. IM 接入
4. cron 调度
5. 远程入口
6. 新一级页面或重型能力域

### 12.2 第 1 周验收项（Stage B）

必须逐项说明：

1. 是否已形成 `输入类型表` 和 `沉淀字段表`。
2. 是否已完成至少 `2` 份真实资料导入样本。
3. 是否已形成至少 `5` 条结构化沉淀样本。
4. `5` 条检索问句是否已有命中结果和未命中原因。
5. `5` 条真实问句回答是否已出现历史沉淀复用痕迹。
6. 是否已完成至少 `1` 次“导入 -> 沉淀 -> 回答 -> 回写”的端到端留证。

填写要求：

1. 每项都必须写清 `做了什么 / 没做什么 / 怎么验证 / 证据在哪`。
2. 没有真实样本或证据路径，默认该项 `不通过`。

第 1 周通过标准：

1. 上述 `6` 项中不得有阻断性缺项。
2. 至少 `1` 条完整链路可以被后续内测复跑。

### 12.3 第 2 周验收项（Stage C + 最小 Stage D）

必须逐项说明：

1. 是否已形成 `状态字段表` 和 `状态更新规则表`。
2. 是否已完成状态更新样本和状态快照。
3. `6` 条续推问题是否完成专项记录。
4. 下一步建议是否已固定最小格式，并有样本留证。
5. 是否已有 `1-2` 类低风险动作接入建议链路。
6. 执行结果是否已回写到 `近期结果 / 阶段结论 / 下一步候选`。
7. 是否已完成一次两周总演练并留证。

填写要求：

1. 每项都必须写清 `做了什么 / 没做什么 / 怎么验证 / 证据在哪`。
2. 如果涉及代码、前端或运行时改动，必须补构建结果或人工核对记录。

第 2 周通过标准：

1. `6` 条续推问题中至少 `4` 条达到“可直接推进下一步”的程度。
2. 至少 `1` 个低风险执行动作已闭合“建议 -> 行动 -> 回写”链路。
3. 两周总演练已形成稳定证据映射。

### 12.4 任务卡验收方法

如果后续不是按天推进，而是按任务卡推进，统一按下面方法验收：

1. `B` 类任务卡：重点看是否增强“导入、沉淀、检索、回写”主链路。
2. `C` 类任务卡：重点看是否增强“状态理解、续推回答、下一步建议”主链路。
3. `D` 类任务卡：重点看是否增强“低风险执行、验证、回写、失败收口”主链路。

每张任务卡都必须回答：

1. `是否有真实样本`
2. `是否进入主链路`
3. `是否有留痕`
4. `是否减少了用户重复劳动`

只要有任一问题无法回答，默认该任务卡 `有条件通过` 或 `不通过`，不能直接宣称完成。

### 12.5 推荐结论写法

后续按两周模板推进时，结论建议统一写成：

1. `通过`
   适用：主链路闭合、样本和证据完整、可复跑。
2. `有条件通过`
   适用：主链路已基本成立，但仍缺单点样本、单点验证或单点回写。
3. `不通过`
   适用：只有描述，没有真实样本；或能力未进入主链路；或无证据路径。
