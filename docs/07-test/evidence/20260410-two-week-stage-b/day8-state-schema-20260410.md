# Stage C Day8 状态字段与更新规则（2026-04-10）

## 1) 最小状态字段表

1. 当前目标：本轮最想达成的结果，来源于用户直接目标句或当轮任务标题。
2. 当前主题：本轮主主题标签，来源于问句主题关键词与近期证据主题。
3. 当前阶段：B/C/D 阶段标签，来源于执行清单和当前证据目录。
4. 最近进展：最近一次 completed+verified 的关键进展。
5. 当前阻塞：当前最影响推进的一条阻塞或风险。
6. 最近结果：最近一次可用结果（answer/verified）摘要。
7. 下一步候选：当前最可执行的一步动作。

## 2) 状态更新规则表（最小）

1. 规则 R1：当 run_finished 且 completion_status=completed 且 verification_code=verified 时，更新“最近进展/最近结果”。
2. 规则 R2：当可用率下降或出现 recovery/system 聚集时，更新“当前阻塞”为最高占比失败类型。
3. 规则 R3：当阶段判断为“继续”时，更新“下一步候选”为该判断给出的第一动作。
4. 规则 R4：当回写记录通过追溯校验时，更新“当前阶段”为可继续推进状态。
5. 规则 R5：当同题复跑达到阈值（>=4/5）时，更新“最近结果”为“可交验候选”。

## 3) Day8 对接要求

1. Day9 必须补一份状态快照样本（字段值实例）。
2. Day10 的 6 条续推问句必须以本表字段作为判定口径。
3. 不新增字段，不改能力域，只做字段值填充与校验。
