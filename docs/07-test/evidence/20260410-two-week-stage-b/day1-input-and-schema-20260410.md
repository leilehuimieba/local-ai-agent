## 1. 输入类型表（Day1 冻结）

| 输入类型 | 典型来源 | 默认落层 | 写入条件 | 不写入条件 | 最小产出 |
|---|---|---|---|---|---|
| 用户目标描述 | 聊天输入、任务描述 | 短期工作记忆 | 能影响当前推进方向 | 仅情绪表达、无任务信息 | `current_goal` |
| 当前计划与步骤 | 用户分步计划、助手分解计划 | 短期工作记忆 | 与当前目标直接相关 | 纯口号、无法执行 | `current_plan` |
| 阻塞与疑问 | 用户卡点、失败反馈 | 短期工作记忆 | 能解释“还差什么” | 与当前目标无关 | `open_issue` |
| 最近执行结果 | 工具调用结果摘要、验证摘要 | 短期工作记忆 | 影响下一步决策 | 全量日志原文、无提炼 | `recent_tool_result` |
| 学习资料原文 | 本地文档、笔记、网页摘录 | 知识沉淀候选 | 有复用价值且可归档来源 | 短时一次性材料 | `document_digest` 候选 |
| 阶段结论 | 任务收口、阶段回顾 | 长期记忆 + 知识库候选 | 具备可复用结论 | 未验证猜测 | `decision_record` |
| 稳定规则 | 用户偏好、项目规则 | 长期记忆 | 跨任务可复用且边界明确 | 临时偏好、一次性限定 | `user_preference` 或 `project_rule` |
| 失败教训 | 复盘后确认的反例 | 长期记忆 | 有明确触发条件和规避动作 | 只描述情绪，不含教训 | `lesson_learned` |
| 架构与模块知识 | 设计文档、模块职责总结 | 本地知识库 | 稳定、去噪、可引用 | 未整理原文堆积 | `architecture_note` / `module_knowledge` |
| 命令与排障模式 | 通过验证的命令流程 | 本地知识库 | 已验证、可重复执行 | 未验证命令片段 | `command_pattern` / `debug_guide` |

## 2. 沉淀字段表（Day1 冻结）

### 2.1 跨层统一字段

| 字段 | 含义 | 必填 |
|---|---|---|
| `id` | 记录唯一标识 | 是 |
| `workspace_id` | 工作区标识 | 是 |
| `layer` | `working_memory` / `long_term_memory` / `knowledge_base` | 是 |
| `entry_type` | 记录类型（如 `project_rule`） | 是 |
| `title` | 记录标题 | 是 |
| `summary` | 记录摘要 | 是 |
| `source` | 来源文件或来源说明 | 是 |
| `source_run_id` | 来源运行 ID（如有） | 否 |
| `tags` | 主题标签 | 否 |
| `verified` | 是否已验证 | 是 |
| `created_at` | 创建时间 | 是 |
| `updated_at` | 更新时间 | 是 |

### 2.2 知识沉淀增强字段

| 字段 | 含义 | 必填 |
|---|---|---|
| `topic` | 当前主题（如“英语学习”“项目收口”） | 是 |
| `conclusion` | 可复用结论 | 是 |
| `evidence` | 依据摘要（不是全量日志） | 是 |
| `next_step` | 下一步动作建议 | 是 |
| `confidence` | 可信度（`low` / `medium` / `high`） | 否 |
| `status` | `candidate` / `confirmed` / `archived` | 是 |

### 2.3 Day1 字段使用规则

1. 任何进入长期层或知识库层的记录，必须同时具备：`conclusion`、`evidence`、`next_step`。
2. `source` 必须可追溯到文件路径或 run_id，不能写“来自历史上下文”这类泛描述。
3. 全量原始日志、工具原始输出默认不写入 `summary`，只能进入 artifact。
4. `verified=false` 的条目可以保留为 `candidate`，但不能作为默认建议依据。

## 3. Day2 对接要求

Day2 默认直接复用本文件，不再重定义输入和字段口径。

Day2 需要完成：

1. 选 `2-3` 份真实资料样本导入。
2. 每条样本至少填齐：`topic`、`summary`、`source`、`conclusion`、`next_step`。
3. 在证据目录补 `导入样本清单` 和 `导入前后对照记录`。
