# 学习续接表达质量验证说明

## 本轮范围

本轮只优化历史页右侧“学习续接”卡片的前端提取逻辑与展示文案收口，不新增页面，不改共享合同，不改后端接口。

## 本轮收紧点

### 1. 降低明显误判

上一轮规则只要文本里出现“学习、掌握、回顾”等词就可能触发学习续接卡片，容易把带有“长期学习”产品表述的项目说明记录误判成学习记录。

本轮新增了一条明确收紧：

1. 只有同时满足“学习意图”与“学习证据”才展示学习续接卡片。
2. `project_answer` 且内容明显是“项目说明 / 需求文档 / 产品定义 / 工作区 / runtime / gateway”类记录时，不再作为学习续接卡片来源。

这条规则主要用于降低“项目说明类记录被当成学习记录”的明显误判风险。

### 2. 提升四个字段的贴合度

本轮不再直接把整段 `summary` / `verification_summary` / `knowledge_digest` 生硬截断，而是改为：

1. `当前学习主题`
   优先从任务标题、摘要、会话摘要里提取带学习意图的短句。
2. `当前掌握情况`
   优先从验证摘要、结果摘要里提取“已理解、已掌握、已整理、能解释”等更像学习进度的表达。
3. `待巩固 / 待补`
   优先提取“待巩固、不稳、薄弱、疑问、易错、补看”等更像学习缺口的表达。
4. `下一步学习建议`
   优先使用现有 `next_step` 中更贴学习动作的句子；没有明确句子时，再基于“待巩固 / 待补”生成一条最小续接建议。

## 前端本地验证样本说明

当前仓库中的真实记录仍以工程开发和项目说明类为主，因此本轮先做“规则对照”，再补真实模拟样本：

1. 负样本：
   `project_answer` 类型、内容包含“项目说明 / 需求文档 / 产品定义 / runtime / gateway”等词的项目说明记录
   预期结果：不再展示学习续接卡片
2. 正样本：
   含“学习 / 复习 / 回顾 / 理解 / 掌握”意图，且在 `verification_summary`、`knowledge_digest`、`memory_digest`、`next_step` 中能看到学习进度或待补线索的记录
   预期结果：继续展示学习续接卡片，且四个字段比上一轮更像真实学习过程提炼
