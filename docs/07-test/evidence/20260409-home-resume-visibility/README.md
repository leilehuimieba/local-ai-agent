# 首页恢复态手动验收说明

## 验收目标

本轮只补首页恢复态的页面级手动证据，用于验证“状态续接可见性增强”已经能在设置正常加载的前端环境中被真实看到。

本轮不包含 provider settings 相关改动的验收放行，也不把工作区里的其他 Go、Rust、设置页改动计入本轮通过依据。

## 触发方式

1. 启动本地链路，确保 gateway 与 runtime 正常运行。
2. 验证设置接口可访问：`http://127.0.0.1:8897/api/v1/settings`
3. 打开首页恢复态预览入口：`http://127.0.0.1:8897/?home_preview=resume`
4. 截取页面级证据：`home-resume-preview.png`

补充记录：

1. `settings-response.json` 保存的是本轮真实 settings 响应。
2. `health-checks.txt` 保存的是本轮 gateway 与 runtime 健康检查结果。
3. `home-real-home.png` 保存的是真实首页路径 `http://127.0.0.1:8897/` 的页面截图，用于说明设置接口已正常加载，首页可进入非阻塞态。

## 页面实际可见内容

`home-resume-preview.png` 中可直接看到以下关键区域：

1. `恢复上下文`
   页面展示最近任务、当前阶段、会话摘要、记忆摘要、知识摘要、思考摘要等续接依据。

2. `从这里继续`
   页面展示下一步线索，并把当前运行、当前会话、最近结果收口等内容组织成可续接入口。

3. `系统与风险`
   页面展示当前系统状态、风险提示或环境提示，用于帮助用户判断是否适合直接继续工程任务。

4. `最近活动`
   页面展示最近运行记录和事件线索，用于帮助用户快速判断最近做到了哪里。

## 关键判定

1. 首页恢复态不是仅存在于代码中，而是在设置接口正常加载的运行环境中被实际看见。
2. 本轮主要证据使用首页恢复态预览入口，是因为该入口复用了现有首页组件和恢复态展示结构，能够稳定呈现本轮新增的“状态续接可见性”信息。
3. 本轮未继续扩展新的展示功能，也未修改 provider settings 能力本身。

## 验收边界

本轮只覆盖以下范围：

1. 首页恢复态中的“恢复上下文 / 从这里继续 / 系统与风险 / 最近活动”四个区域的页面级可见证据。
2. “状态续接可见性增强”相关展示是否已经具备手动复核材料。

本轮明确不计入以下内容：

1. provider settings 相关工作区改动。
2. 工作区中其他 Go、Rust、执行器拆分、catalog、凭据管理相关改动。
3. 学习场景、日志页学习视角、下一阶段总体验收结论。

## 验证结果

1. 首页恢复态四个关键区域已具备截图证据。
2. 设置接口、gateway、runtime 已有响应留档，可证明截图不是离线拼接结果。
3. 本轮补证据过程中未发现必须新增代码修复的问题。
4. 前端展示增强的代码构建结果已通过一次 `frontend` 目录下的 `npm run build`，本轮补证据未新增前端实现改动。
