# 任务页收口样式隔离说明

## 说明目的

本文件用于把“本轮任务页体验收口真正依赖的样式依据”与工作区内其他样式脏改动分开，避免验收时把首页样式或其他入口样式误计入本轮通过依据。

## 本轮真正依赖的样式点

本轮任务页页面证据真正依赖的样式点，集中在任务页主消息区和过程条带两个区域。

### 1. 结果头部标签收口

对应类：

- `.thread-record-meta`

作用：

- 让 `正式回答 / 恢复结果 / 系统说明` 的角色标签与状态标签能并列展示
- 支撑页面级证据中“用户一眼看出当前结果类型”的视觉分层

这部分是任务页回答分层成立的直接样式依据。

### 2. 过程事件条带收口

对应类：

- `.tool-event-strip`

作用：

- 把原本较散的过程 badge 包进一个单独的“当前进度”区块
- 让工具过程展示从满屏噪音收紧为次级信息

这部分是“badge 洪流明显收口”的直接样式依据。

## 可作为弱关联支撑，但不是本轮主证据核心的样式

以下样式虽然可能参与了右栏或上下文卡片的排布，但不是本轮放行的核心样式证据：

- `.context-highlight-grid`
- `.context-highlight-card`

原因：

- 本轮页面级放行主要看的是右栏结构已收紧、repo 主区块已移出
- 这些类更多是局部信息卡片排布优化，不应替代主证据

## `frontend/src/index.css` 中应计入本轮依据的 diff

当前 `index.css` diff 中，与任务页体验收口直接相关、可计入本轮通过依据的主要是：

1. `.thread-record-meta`
2. `.tool-event-strip`

如果验收需要从样式层进一步解释页面变化，可以把这两块视为本轮最小有效样式依据。

## 不应计入本轮通过依据的混入样式

以下内容虽然也出现在 `frontend/src/index.css` diff 中，但不应作为“任务页体验收口”放行依据：

### 1. 首页相关样式

- `.home-scene`
- `.home-hero-section`
- `.home-card-section`
- `.home-block-section`
- `.home-grid`
- `.home-grid-two`
- `.home-compose-*`
- `.home-example-list`
- `.home-metric-grid`
- `.home-capability-list`
- `.home-resume-*`
- `.topbar-summary`

这些属于首页恢复态或首页布局相关样式，不是本轮任务页证据的核心范围。

### 2. 学习续接相关样式

- `.learning-continuation-card`
- `.learning-continuation-list .sidebar-row`

这些属于通用学习场景的历史收口内容，不应混入本轮任务页放行依据。

### 3. 响应式与布局混入项

部分 media query 中对首页网格、首页按钮区的补充，也不应计入本轮任务页通过依据。

## 验收使用建议

验收 AI 在判断本轮是否成立时，建议优先依据：

1. 页面截图中主消息头部是否明确区分 `正式回答 / 恢复结果 / 系统说明`
2. 页面截图中过程区是否已经收紧为 `当前进度`
3. 右栏主结构是否已不再出现 repo 主区块

不要把首页样式扩展或其他历史脏改动，误判为本轮任务页收口必须成立的前提条件。
