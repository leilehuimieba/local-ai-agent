更新时间：2026-04-02

状态：已完成实现，待最终人工复核。

## 1. 本轮目标

按任务书要求，把运行时“动作分支”收口为统一工具协议，并建立运行时内的 Tool Registry 入口。

## 2. 实现范围

1. 新增 `ToolRegistry`
2. 新增 `ToolCall / ToolSpec / ToolName`
3. 保留现有 `PlannedAction` 作为当前阶段的 planner 输出基础
4. 将主链路改为由 `ToolRegistry` 负责可见工具和当前 `ToolCall`

## 6. 运行链路变化

调整前：

1. 工具可见性和动作分支虽然存在，但没有 `ToolRegistry` 正式落点
2. 主循环直接围绕 `PlannedAction` 展开

调整后：

1. `runtime_tool_registry()` 提供统一入口
2. `ToolRegistry.visible_tools()` 负责当前模式下工具过滤
3. `ToolRegistry.plan_tool_call()` 负责生成当前 `ToolCall`
4. 主循环事件 metadata 已回流 `input_schema / requires_confirmation`
