## 5. 运行链路变化

调整前：

1. `planner` 主要直接依赖用户输入和会话摘要
2. prompt 在 `execution.rs` 内局部拼接
3. 上下文入口分散

调整后：

1. `bootstrap_run` 先构建 `RuntimeContextEnvelope`
2. `ToolRegistry` 基于上下文决定当前 `ToolCall`
3. `prompt.rs` 负责统一渲染 `ContextAnswer / ProjectAnswer` prompt
