# 缺口检查清单

按下面顺序检查：

1. 当前执行入口是否唯一且明确。
2. 当前阶段、阶段目标和 Gate 是否明确。
3. `changes/INDEX.md` 是否标记了当前活跃 change。
4. 是否存在进行中的 change 目录。
5. `proposal.md` 是否说明目标、非目标、验收口径。
6. `design.md` 是否说明模块影响、contract 变化、风险与回退。
7. `tasks.md` 是否拆成可验证的小任务，并写清完成判据。
8. `status.md` 是否反映真实进度、阻塞点和下一步。
9. `verify.md` 是否提供测试方式、证据位置和 Gate 映射。
10. 当前请求是否试图越过缺口直接进入大规模实现。
