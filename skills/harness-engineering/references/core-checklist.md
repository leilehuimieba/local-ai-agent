# 核心清单

这是通用版顺序，跨项目都适用。

## 1. 主循环

至少回答：

1. 任务怎样开始
2. 怎样形成计划
3. 怎样执行
4. 怎样观察结果
5. 怎样验证
6. 怎样结束或继续

没有显式主循环时，优先补主循环。

## 2. 上下文装配

只给“下一步所需最小包”：

1. 当前目标
2. 当前状态
3. 相关历史摘要
4. 必要工具预览
5. 必要约束

避免：

1. 全量注入
2. 长文档原文直塞
3. 大日志直接回灌

## 3. 工具合同

每个工具都应有：

1. 名称
2. 输入
3. 输出
4. 风险
5. 是否需要确认
6. 可否重试

## 4. 权限与确认

确认应：

1. 前置
2. 可恢复
3. 有原因
4. 可审计

## 5. 验证闭环

至少补一类外部证据：

1. 测试
2. 规则检查
3. 运行观测
4. 人工确认

## 6. Artifact 外置

大输出只保留：

1. summary
2. preview
3. path 或 handle

## 7. 记忆与知识路由

只做：

1. 相关召回
2. 有价值沉淀
3. 摘要回填

不做：

1. 默认全读
2. 默认全写

## 8. 长任务交接

交接包至少有：

1. 当前目标
2. 已完成项
3. 未决风险
4. 下一步
5. 关键 artifact
