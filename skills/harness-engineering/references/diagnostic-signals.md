# 诊断信号

先判断失败像什么，再决定补哪里。

## 1. 上下文失稳

常见信号：

1. 给的信息越多，输出越差。
2. 后半段开始忽略前文关键约束。
3. 同一条规则前面遵守，后面忘掉。
4. 临近上下文上限时开始急着结束。

优先补：

1. 上下文装配
2. 摘要与压缩
3. handoff artifact

## 2. 工具失稳

常见信号：

1. 模型反复误用工具。
2. 工具调用结果不可预测。
3. 不同工具返回风格完全不同。
4. 失败后模型不知道能否重试。

优先补：

1. 工具协议
2. 统一结果结构
3. 风险级别与确认位

## 3. 验证失稳

常见信号：

1. 生成者总觉得“差不多完成了”。
2. 一改再改，但没有明确通过标准。
3. 修复后引入回归没人发现。

优先补：

1. verify 阶段
2. 独立 evaluator
3. 自动化和回归证据

## 4. 权限失稳

常见信号：

1. 高风险动作被直接执行。
2. 用户确认后任务要从头再来。
3. 审批原因不透明。

优先补：

1. permission gate
2. 确认恢复机制
3. 风险事件

## 5. 长任务失稳

常见信号：

1. 前半程计划清楚，后半程开始漂移。
2. 中断后无法恢复现场。
3. 模型为了尽快结束而压缩质量。

优先补：

1. 分段推进
2. 交接包
3. 状态重建
